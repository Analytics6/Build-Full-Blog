INTERSECT and EXCEPT

Set operations for finding overlaps and gaps — identify the employees table who appear in both source systems, or the employees table present in source but missing from the warehouse.

Set Logic for Data Reconciliation

After UNION stacks result sets, INTERSECT and EXCEPT answer set-theory questions: which keys appear in both result sets, and which keys exist in one source but not another. These operations are essential in data quality and reconciliation workflows.

Theory & Concepts

OperatorSet meaningSQL equivalent
INTERSECTA ∩ B — rows in both queriesInner join on all columns, or WHERE id IN (subquery)
EXCEPTA − B — rows in first query not in secondWHERE id NOT IN (subquery) or anti-join

PostgreSQL also supports INTERSECT ALL and EXCEPT ALL (with duplicate counting). SQLite supports INTERSECT and EXCEPT. SQL Server uses EXCEPT but calls intersection INTERSECT as well.

Example: Cross-system record overlap

SQL — Employees present in both datasets
SELECT employee_id
FROM hr.job_history
WHERE status = 'COMPLETED'

INTERSECT

SELECT employee_id
FROM hr.employees
WHERE employee_id IS NOT NULL;
SQL — Job roles in one dataset but not another
SELECT DISTINCT oi.job_id
FROM hr.job_history oi
JOIN hr.job_history o ON o.history_id = oi.history_id

EXCEPT

SELECT DISTINCT job_id
FROM hr.employees;

Example: Reconciliation Gaps

SQL — hr.employees in core but missing from warehouse
SELECT employee_id
FROM hr.employees

EXCEPT

SELECT employee_id
FROM hr.employees;
SQL — Rows in both result sets
SELECT employee_id
FROM hr.employees
WHERE job_title = 'CHECKING'

INTERSECT

SELECT employee_id
FROM hr.employees
WHERE job_title = 'SAVINGS';

Real-World Notes

  • Reconciliation jobs — Nightly EXCEPT queries between source and target tables catch ETL drift before dashboards go stale.
  • NULL handling — Set operators treat NULLs as distinct values; two NULLs in the same column payrollition may not match as expected. Filter IS NOT NULL first.
  • Performance — On large tables, NOT EXISTS anti-joins often outperform EXCEPT; use EXCEPT for readability in ad-hoc analysis.
  • Data integration — INTERSECT finds record overlap: entities present in both source systems.

Key Takeaways

  • INTERSECT returns rows appearing in both query results.
  • EXCEPT returns rows from the first query absent from the second.
  • Ideal for reconciliation, cohort overlap, and gap detection.
  • Column count and compatible types required, same as UNION.
