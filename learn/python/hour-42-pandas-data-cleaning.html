Cleaning Dirty Data

Real store_sales and transactions feeds arrive with missing values, duplicates, and inconsistent column names. Pandas data cleaning — handling NaN, deduplicating, imputing, renaming — is the critical step between raw ingest and trustworthy data analytics.

Theory & Concepts

Missing Data (NaN)

Pandas represents missing values as NaN. Audit with df.isnull().sum(). Impute with fillna (mean/median/mode) or remove with dropna.

Duplicates

Duplicate rows inflate counts — use drop_duplicates(subset=[...]) on natural keys like transaction_id.

Column Renaming

Standardize to snake_case with df.rename(columns={...}) before SQL loads.

Syntax Reference

isnull().sum() counts missing values per column — the first audit on any freshly loaded dataset.

isnull().sum() — audit missing values
import pandas as pd

df = pd.read_csv("transactions.csv")

print(df.isnull().sum())
# Output (example):
# transaction_id      0
# account_id          3
# amount             12

fillna() replaces missing values — commonly with the column median or mean. Assign the result back rather than relying on inplace.

fillna() — impute missing values
median_amount = df["amount"].median()

df["amount"] = df["amount"].fillna(median_amount)

print(df["amount"].isnull().sum())    # Output: 0 — no missing amounts left

dropna() removes rows with missing values — use subset= to require only the columns that truly matter.

dropna() — remove incomplete rows
before = len(df)

df = df.dropna(subset=["account_id"])    # a txn without an account is unusable

print(before - len(df))    # Output: 3 — rows removed

duplicated() flags repeat rows; drop_duplicates() removes them — always on a natural key like the transaction ID.

drop_duplicates() — deduplicate on a key
print(df.duplicated(subset=["transaction_id"]).sum())
# Output (example): 5 — duplicate transaction IDs found

df = df.drop_duplicates(subset=["transaction_id"])

print(df.duplicated(subset=["transaction_id"]).sum())   # Output: 0

rename() standardizes column names — map old names to new ones before a database load.

rename() — standardize column names
df = df.rename(columns={"txn_type": "transaction_type", "Amt": "amount"})

print(list(df.columns))
# Output: ['transaction_id', 'account_id', 'amount', 'transaction_type']

astype() converts a column's type — essential when numbers were read as text.

astype() — convert column types
df["amount"] = df["amount"].astype(float)

print(df["amount"].dtype)    # Output: float64 — math now behaves

Real-World Application

  • Inventory data — Fill missing unit_price with category median before valuation.
  • Data systems — Deduplicate on transaction_id after merging multiple feed sources.
  • signups — Standardize column names before loading to PostgreSQL.

Notebook & Practice

  • Remove duplicate entries from a customer signup list.
  • Rename columns to snake_case style.
  • Compare row counts before and after dropna() vs fillna().

Key Takeaways

  • .isnull().sum() audits missing data.
  • .fillna() imputes; .dropna() removes incomplete rows.
  • .drop_duplicates() prevents inflated aggregations.
