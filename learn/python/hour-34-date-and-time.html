Date and Time

Subtopics: datetime class, date, time, strftime (formatting dates), strptime (parsing dates), timedelta (date calculations)

Go to Assignment →

Why Date & Time Matter in Data Analytics

Every structured dataset is time-stamped: when a POS sale posted, when a transfer cleared, when a transfer cleared. The datetime module lets you create timestamps, format them for reports, parse messy date strings from CSV feeds, and compute durations with timedelta — essential for SLA tracking and incremental ETL loads.

Theory & Concepts

Core Classes

  • datetime — date + time combined (e.g., 2026-07-15 14:30:00)
  • date — date only
  • time — time only
  • timedelta — duration for addition/subtraction

strftime vs strptime

  • strftime — datetime — formatted string (for reports/logs)
  • strptime — string — datetime (for parsing CSV date columns)

Common Format Codes

Syntax Reference

datetime.now() captures the current moment; datetime(...) builds a specific timestamp from components.

datetime.now() — creating timestamps
from datetime import datetime

now = datetime.now()
print(now)              # Output: e.g. 2026-07-15 14:30:00.123456
print(now.year)         # Output: 2026 — components are attributes

run_start = datetime(2026, 7, 15, 2, 0, 0)   # build a specific moment
print(run_start)        # Output: 2026-07-15 02:00:00

strftime() turns a datetime into a formatted string — for reports, filenames, and logs. The format codes choose the layout.

strftime() — datetime to formatted string
from datetime import datetime

moment = datetime(2026, 7, 15, 14, 30, 0)

print(moment.strftime("%Y-%m-%d %H:%M:%S"))   # Output: 2026-07-15 14:30:00
print(moment.strftime("%A, %B %d, %Y"))       # Output: Wednesday, July 15, 2026
print(moment.strftime("%d/%m/%Y"))            # Output: 15/07/2026

strptime() goes the other way: it parses a string into a datetime, using a format string that must match the text exactly.

strptime() — string to datetime
from datetime import datetime

raw = "15/07/2026 09:45:00"     # a CSV date column, day-first format

txn_time = datetime.strptime(raw, "%d/%m/%Y %H:%M:%S")

print(txn_time)         # Output: 2026-07-15 09:45:00
print(txn_time.year)    # Output: 2026 — now a real datetime, not text

timedelta represents a duration. Add or subtract it from datetimes to schedule and shift windows.

timedelta — date arithmetic
from datetime import datetime, timedelta

last_run = datetime(2026, 7, 10, 2, 0, 0)

next_run = last_run + timedelta(days=1)
print(next_run.date())          # Output: 2026-07-11

one_week_ago = last_run - timedelta(weeks=1)
print(one_week_ago.date())      # Output: 2026-07-03

Subtracting two datetimes yields a timedelta — the standard way to measure elapsed time or days remaining.

Datetime subtraction — measuring gaps
from datetime import datetime

initiated = datetime(2026, 7, 15, 9, 45)
settled = datetime(2026, 7, 15, 14, 30)

gap = settled - initiated
print(gap)                              # Output: 4:45:00
print(gap.total_seconds() / 3600)       # Output: 4.75 — hours to settle

remaining = datetime(2026, 12, 31) - datetime(2026, 7, 15)
print(remaining.days)                   # Output: 169 — days to year end

Real-World Application

  • Incremental ETL — Load only rows where updated_at > last_run using datetime comparisons.
  • Data analytics — Group sales by week using strftime("%Y-W%W").
  • Service SLAs — Compute hours between transaction initiation and settlement with timedelta.

Key Takeaways

  • datetime.now() captures the current moment; timedelta handles date arithmetic.
  • strptime parses strings; strftime formats datetime for output.
  • Consistent date parsing prevents silent ETL bugs from ambiguous formats.
  • Timestamps are the backbone of incremental loads and audit trails in analytics.
