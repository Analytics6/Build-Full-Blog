Transactions and ACID

COMMIT, ROLLBACK, and data integrity — ensure salary updates and job_history inserts either fully succeed or fully fail.

Go to Assignment →

All-or-Nothing Data Changes

Updating related rows often requires multiple statements: change a value in one table, then insert a matching history row. If only the first statement succeeds, the database ends up in an inconsistent state. Transactions wrap multiple statements into one atomic unit — either every change commits together or none do.

Theory: The ACID Properties

PropertyMeaningExample
AtomicityAll or nothingBoth salary update and job_history insert succeed, or neither runs
ConsistencyValid state before and afterEmployee record matches job history after promotion
IsolationConcurrent transactions don't interfereTwo promotions don't read partial updates
DurabilityCommitted data survives crashesAfter COMMIT, promotion persists after reboot

Transaction Control Statements

SQL — Employee promotion with explicit transaction
BEGIN;

UPDATE hr.employees
SET job_id = 'SA_REP',
    salary = 9500,
    department_id = 80
WHERE employee_id = 143
  AND salary < 9500;

INSERT INTO hr.job_history (employee_id, start_date, job_id, department_id)
VALUES (143, CURRENT_DATE, 'SA_REP', 80);

COMMIT;

Example: Department Transfer with Audit Trail

SQL — Transfer with job history log
BEGIN;

UPDATE hr.employees
SET department_id = 90,
    job_id = 'IT_PROG'
WHERE employee_id = 201;

INSERT INTO hr.job_history (employee_id, start_date, end_date, job_id, department_id)
VALUES (201, CURRENT_DATE, NULL, 'IT_PROG', 90);

COMMIT;

When to ROLLBACK

If a validation check fails mid-transaction, ROLLBACK undoes every change since BEGIN.

SQL — Abort on failed budget check
BEGIN;

UPDATE hr.employees
SET salary = salary + 5000
WHERE employee_id = 101;

-- Validation failed; undo the salary change
ROLLBACK;

Key Takeaways

  • Transactions group statements into one atomic unit.
  • ACID: Atomicity, Consistency, Isolation, Durability.
  • COMMIT saves changes; ROLLBACK undoes them.
  • Critical for promotions, job_history updates, and reliable ETL.
