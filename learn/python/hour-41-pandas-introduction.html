Pandas: The Data Scientist's Workbench

Pandas brings SQL-table semantics to Python. A DataFrame is a labeled 2D table — like store_sales or transactions loaded into memory. Analysts use Pandas for exploration, cleaning, and aggregation before forecasting product demand or detecting fraud detection.

Theory & Concepts

  • Series — 1D labeled array (one column).
  • DataFrame — 2D table with named columns and an index.
  • .head(), .info(), .describe() — first steps on any new dataset.
  • pd.read_csv() — load CSV files directly into DataFrames.

Syntax Reference

A Series is one labeled column. Creating one from a list gives it a default integer index.

pd.Series — one labeled column
import pandas as pd

prices = pd.Series([49.99, 299.00, 24.99])

print(prices.mean())    # Output: 124.66
print(prices[0])        # Output: 49.99 — index label 0

A DataFrame built from a dict: keys become column names, lists become column values.

pd.DataFrame — table from a dict
import pandas as pd

products = pd.DataFrame({
    "product_id": ["P-001", "P-002", "P-003"],
    "product_name": ["Keyboard", "Monitor", "Mouse"],
    "unit_price": [49.99, 299.00, 24.99],
})

print(products)
# Output:
#   product_id product_name  unit_price
# 0      P-001     Keyboard       49.99
# 1      P-002      Monitor      299.00
# 2      P-003        Mouse       24.99

pd.read_csv() loads a CSV straight into a DataFrame — with type inference, header detection, and quoting handled for you.

pd.read_csv() — load a CSV file
import pandas as pd

txns = pd.read_csv("transactions.csv")

print(txns.shape)               # Output: e.g. (14230, 5) — rows, columns
print(txns["amount"].mean())    # Output: e.g. 1263.42

head() and tail() preview the first and last rows — always the first call on a fresh dataset.

head() / tail() — preview rows
print(products.head(2))
# Output:
#   product_id product_name  unit_price
# 0      P-001     Keyboard       49.99
# 1      P-002      Monitor      299.00

print(products.tail(1))
# Output:
#   product_id product_name  unit_price
# 2      P-003        Mouse       24.99

info() summarizes dtypes and null counts; describe() computes statistics for numeric columns — together, an instant data-quality snapshot.

info() / describe() — first-look EDA
products.info()
# Output (abridged):
# RangeIndex: 3 entries, 0 to 2
# unit_price    3 non-null   float64

print(products["unit_price"].describe())
# Output (abridged):
# count      3.00
# mean     124.66
# min       24.99
# max      299.00

Real-World Application

  • ETL — Validate CSV schema and null rates before warehouse load.

Key Takeaways

  • DataFrame = in-memory SQL table with labeled columns.
  • .head(), .info(), .describe() for initial EDA.
  • pd.read_csv() is the primary ingest path for tabular files.
