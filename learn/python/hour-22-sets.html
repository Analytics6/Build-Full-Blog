Go to Assignment →

Unique Values, Fast Lookups

A set is an unordered collection of unique elements. Add a duplicate and nothing happens — the set already has it. Two properties make sets indispensable: automatic deduplication, and near-instant (O(1)) membership testing, no matter how large the set grows. Sets also support the mathematical operations you know from Venn diagrams: union, intersection, and difference.

Theory & Concepts

  • Creation — {1, 2, 3} or set(iterable). Careful: {} creates an empty dict; the empty set is set().
  • Unordered — sets have no indices; s[0] is a TypeError. Iterate or convert to a list when order matters.
  • Hashable elements only — numbers, strings, and tuples are fine; lists and dicts cannot go into a set.

Creating Sets

Building a set from a list silently drops duplicates — the fastest deduplication tool in Python.

set() — create and deduplicate
txn_ids = ["TXN-1", "TXN-2", "TXN-1", "TXN-3", "TXN-2"]

unique_ids = set(txn_ids)
print(unique_ids)         # Output: {'TXN-1', 'TXN-2', 'TXN-3'} — order may vary
print(len(unique_ids))    # Output: 3 — two duplicates removed

empty = set()             # NOT {} — that would be an empty dict
print(type(empty))        # Output: <class 'set'>

Adding & Removing Elements

add(item) inserts one element. Adding something already present changes nothing — no error, no duplicate.

set.add() — insert one element
fraud_accounts = {"ACC-999", "ACC-888"}

fraud_accounts.add("ACC-777")
fraud_accounts.add("ACC-999")      # already present — silently ignored

print(len(fraud_accounts))         # Output: 3

update(iterable) adds every element of another collection at once — the set version of extend().

set.update() — add many at once
blocklist = {"ACC-999"}

blocklist.update(["ACC-888", "ACC-777", "ACC-999"])

print(sorted(blocklist))
# Output: ['ACC-777', 'ACC-888', 'ACC-999'] — the duplicate collapsed

remove(item) deletes an element and raises KeyError if it is missing; discard(item) deletes without complaining. Choose based on whether a missing element is a bug.

set.remove() / set.discard() — delete an element
blocklist = {"ACC-999", "ACC-888"}

blocklist.remove("ACC-888")        # present — removed
blocklist.discard("ACC-000")       # absent — no error
# blocklist.remove("ACC-000")      <-- KeyError: 'ACC-000'

print(blocklist)                   # Output: {'ACC-999'}

pop() removes and returns an arbitrary element (sets have no order, so there is no "last"); clear() empties the set.

set.pop() / set.clear() — arbitrary removal, empty all
pending_reviews = {"ACC-101", "ACC-102", "ACC-103"}

next_case = pending_reviews.pop()     # some element — you cannot choose which
print(next_case)                      # Output: e.g. ACC-102

pending_reviews.clear()
print(pending_reviews)                # Output: set()

Set Operations

union() (or |) combines two sets into one containing every element from both.

set.union() — all elements from both
store_a = {1001, 1002, 1003, 1004}
store_b = {1003, 1004, 1005, 1006}

all_customers = store_a | store_b          # same as store_a.union(store_b)
print(sorted(all_customers))
# Output: [1001, 1002, 1003, 1004, 1005, 1006]

intersection() (or &) keeps only the elements present in both sets — e.g. customers who shopped at both stores.

set.intersection() — common to both
store_a = {1001, 1002, 1003, 1004}
store_b = {1003, 1004, 1005, 1006}

loyal_both = store_a & store_b       # same as store_a.intersection(store_b)
print(sorted(loyal_both))            # Output: [1003, 1004]

difference() (or -) keeps elements in the first set that are not in the second. Order matters: a - b and b - a differ.

set.difference() — in one but not the other
store_a = {1001, 1002, 1003, 1004}
store_b = {1003, 1004, 1005, 1006}

only_a = store_a - store_b
only_b = store_b - store_a

print(sorted(only_a))   # Output: [1001, 1002]
print(sorted(only_b))   # Output: [1005, 1006]

symmetric_difference() (or ^) keeps elements in either set but not in both — the "exactly one store" customers.

set.symmetric_difference() — either, not both
store_a = {1001, 1002, 1003, 1004}
store_b = {1003, 1004, 1005, 1006}

one_store_only = store_a ^ store_b
print(sorted(one_store_only))
# Output: [1001, 1002, 1005, 1006]

Set Relationship Tests

issubset() asks "is every element of this set inside that one?"; issuperset() asks the reverse.

set.issubset() / set.issuperset() — containment tests
premium_customers = {1003, 1004}
all_customers = {1001, 1002, 1003, 1004}

print(premium_customers.issubset(all_customers))    # Output: True
print(all_customers.issuperset(premium_customers))  # Output: True
print(all_customers.issubset(premium_customers))    # Output: False

isdisjoint() returns True when two sets share no elements — a clean way to verify that, say, no active account is on the fraud blocklist.

set.isdisjoint() — no overlap at all?
active_accounts = {"ACC-101", "ACC-102"}
fraud_blocklist = {"ACC-999", "ACC-888"}

print(active_accounts.isdisjoint(fraud_blocklist))   # Output: True — all clear
print(active_accounts.isdisjoint({"ACC-101"}))       # Output: False — overlap found

A frozenset is an immutable set: no add or remove after creation. Because it is hashable, it can be a dictionary key or an element of another set.

frozenset — an immutable set
weekend_days = frozenset(["Sat", "Sun"])

print("Sat" in weekend_days)     # Output: True — lookups still work
# weekend_days.add("Fri")        <-- AttributeError: no add on frozenset

# Hashable, so it can key a dict:
schedules = {weekend_days: "reduced staff"}
print(schedules[weekend_days])   # Output: reduced staff

Edge Cases & Gotchas

Membership testing is where sets earn their keep: checking in against a list scans every element, while a set answers in constant time.

Fast membership — set vs list lookups
fraud_accounts = {"ACC-999", "ACC-888"}     # set: O(1) lookup
account = "ACC-101"

if account in fraud_accounts:
    print("Block transaction")
else:
    print("Approve transaction")

# Output: Approve transaction
# With a million-entry blocklist, a list would scan all entries;
# the set answers instantly.

Real-World Application

  • ETL — set() deduplicates customer IDs before a warehouse load.
  • Fraud screening — blocklists live in sets for constant-time checks on every transaction.
  • Analytics — intersection finds cross-store customers; difference finds churned ones between two periods.

Key Takeaways

  • Sets hold unique, hashable elements; set() is the empty set, not {}.
  • Add with add/update; remove with remove (raises), discard (silent), pop (arbitrary), clear.
  • Set math: | union, & intersection, - difference, ^ symmetric difference.
  • issubset/issuperset/isdisjoint answer containment questions in one call.
  • Membership tests are O(1) — use sets for blocklists and dedup, and frozenset when immutability or hashability is needed.
