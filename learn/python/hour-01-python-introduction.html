Why Learn Python?

Welcome to the 45-Hour Python Sprint. Python is one of the most widely used programming languages in the world, and it dominates data analytics, automation, and scripting. Its greatest strength is readability: Python code often reads almost like plain English, which means you spend your energy on solving problems rather than fighting syntax.

By the end of this hour you will understand what Python is, how it came to be, where it is used, and how to set up a Jupyter environment so you can run every example in this course yourself.

Theory & Concepts

What is Python?

Python is an interpreted, high-level, general-purpose programming language created by Guido van Rossum and first released in 1991. Four properties define how it behaves:

  • Interpreted — Python executes your source code line by line through an interpreter. There is no separate compile step, which makes experimentation fast: type a line, run it, see the result.
  • Dynamically typed — You never declare a variable's type. Python figures out at runtime whether a value is a number, text, or something else. This speeds up writing code, but it also means type errors surface at runtime rather than compile time — a trade-off you will learn to manage.
  • Garbage-collected — Memory is managed automatically. When no variable refers to an object any more, Python reclaims the memory for you.
  • Multi-paradigm — Python supports procedural code (simple top-to-bottom scripts), object-oriented programming (classes, covered in Hours 35–38), and functional patterns (lambdas and comprehensions, Hour 28).

A Brief History

Python 1.0 arrived in 1994. Version 2.x dominated the 2000s, and Python 3 (released in 2008) cleaned up long-standing inconsistencies — most famously turning print from a statement into a function. Python 2 reached end of life in 2020; everything you learn in this course is modern Python 3. The name comes from the comedy group Monty Python, not the snake — though the snake logo stuck anyway.

Setting Up Your Environment

Install Anaconda (a Python distribution that bundles data libraries) or the lighter Miniconda, then launch Jupyter Notebook or JupyterLab. Jupyter is the standard workspace for data work because it mixes two kinds of cells:

  • Markdown cells — formatted notes, headings, and explanations.
  • Code cells — executable Python whose output appears directly beneath the cell.
ShortcutAction
Shift + EnterRun cell and move to next
Ctrl + EnterRun cell, stay in place
M (command mode)Convert cell to Markdown
Y (command mode)Convert cell to Code

Common pitfall: beginners often type Python code into a Markdown cell and wonder why nothing runs. Check the cell type in the toolbar — code cells show an In [ ]: prompt on the left.

Syntax Reference

Your first Python program prints text to the console with the built-in print() function. Comments start with # and are ignored by the interpreter — use them to explain why the code does what it does.

Example — Your first Python script
# A first script for a retail analytics team
# print() writes text to the console — the simplest form of output
print("Hello, retail analytics!")
print("Starting store_sales ETL pipeline...")

# Output:
# Hello, retail analytics!
# Starting store_sales ETL pipeline...
Example — Python as a calculator
# Quick arithmetic checks on daily store numbers
stores_reporting = 125        # stores that sent sales files today
avg_transactions = 8          # average transactions per store per hour

print(stores_reporting * avg_transactions)   # Output: 1000
print(stores_reporting + 25)                 # Output: 150
print(stores_reporting / 5)                  # Output: 25.0 — division gives a float
Example — Markdown vs code cells in Jupyter
# In a MARKDOWN cell (not a code cell), you would write notes like:
#
#   # My Learning Goals
#   - Build data pipelines for retail inventory
#   - Analyze monthly store_sales trends
#
# In a CODE cell, you write executable Python:
account_balance = 5000.00
print(f"Opening balance: {account_balance}")   # Output: Opening balance: 5000.0

Real-World Application

Python sits at the center of most modern data workflows:

  • ETL — Orchestration tools like Apache Airflow are written in and scripted with Python; nightly jobs ingest files, transform rows, and load warehouses.
  • Analytics — Pandas (Hours 41–43) turns raw CSVs into aggregated insights in a few lines of code.
  • Services — Frameworks like FastAPI expose data through web APIs, backed by the same Python skills you start learning today.

Notebook & Practice

  • Create a Markdown cell with your name as an H1 heading and a bulleted list of your learning goals.
  • Write print('Welcome to Python programming!') in a code cell and run it with Shift + Enter.
  • Calculate 125 * 8 in a code cell and confirm the output is 1000.
  • Add a # comment above each cell explaining what it does.

Key Takeaways

  • Python is an interpreted, dynamically typed, garbage-collected language designed for readability.
  • Python 3 is the modern standard; Python 2 is end-of-life and should not be used.
  • Jupyter Notebook combines Markdown notes and executable code cells in one workspace — ideal for learning and data analysis.
  • print() and # comments are your first tools for producing output and documenting intent.
  • Run cells with Shift + Enter; check the cell type if code refuses to execute.
