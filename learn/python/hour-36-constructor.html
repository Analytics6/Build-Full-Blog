Constructor

Subtopics: __init__() method, instance attributes, self keyword, default values in parameters

Go to Assignment →

Why Constructors Matter in Data Analytics

In Hour 35 you set attributes manually after creating objects. In production, every Transaction or BankAccount should start in a valid state. The __init__ constructor runs automatically when you instantiate a class, initializing attributes through parameters — so Transaction("TXN-001", 1500.00) is ready to use immediately.

Theory & Concepts

The __init__ Method

A special "dunder" method that Python calls when you write ClassName(...). It sets up instance attributes so every object starts with consistent, valid data.

Understanding self

self refers to the current instance. Use it to assign attributes: self.name = name. Python passes self automatically — never include it when calling the constructor.

Default Parameter Values

Constructors can define defaults: def __init__(self, name, stock=10). Callers omit stock to use 10 — common for optional metadata fields in product catalog records.

Syntax Reference

The constructor: __init__ receives the arguments you pass at instantiation and stores them on self. Every new object arrives fully initialized.

__init__ — initializing at creation
class BankAccount:
    def __init__(self, account_id, balance):
        self.account_id = account_id      # parameter -> instance attribute
        self.balance = balance

acct = BankAccount("ACC-8842", 1500.00)   # __init__ runs automatically

print(acct.account_id)    # Output: ACC-8842
print(acct.balance)       # Output: 1500.0

self is the object being built. self.name = name copies the parameter into the instance — without self, the value would vanish when the constructor ends.

self — the current instance
class Customer:
    def __init__(self, name):
        self.name = name          # stored on the object — survives
        greeting = "hello"        # local variable — gone after __init__

c1 = Customer("Priya")
c2 = Customer("James")

print(c1.name)    # Output: Priya — each instance keeps its own name
print(c2.name)    # Output: James

Default parameter values make constructor arguments optional — callers supply only what differs from the common case.

Defaults in __init__ — optional fields
class Product:
    def __init__(self, name, price, stock=10):
        self.name = name
        self.price = price
        self.stock = stock

notebook = Product("Spiral Notebook A5", 4.50)          # stock defaults to 10
pen_set = Product("Pen Set", 5.99, stock=50)            # override by keyword

print(notebook.stock)    # Output: 10
print(pen_set.stock)     # Output: 50

Real-World Application

  • Database connectors — Constructor opens connection pool with credentials from config.
  • Data models — Transaction(txn_id, amount, status) validates fields at creation time.
  • ETL records — Each parsed CSV row becomes a validated object before load.

  • Instantiate two Employee objects and print their distinct attribute values.
  • Build a BankTransaction constructor requiring txn_id and amount.

Key Takeaways

  • __init__ initializes objects automatically at creation time.
  • self refers to the current instance — use it to set attributes.
  • Default parameter values make constructors flexible for optional fields.
  • Constructors ensure every domain object starts in a valid state.
