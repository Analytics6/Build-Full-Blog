Yes, No, and Missing

Programs answer yes/no questions constantly: is this value above the threshold, does this record exist, has this flag been set? Booleans (True and False) carry those answers. A separate concept — None — represents the absence of a value, the Python equivalent of SQL's NULL. Confusing "no" with "not provided" is a classic source of bugs, so this hour treats them carefully and separately.

Theory & Concepts

The bool Type

True and False (capitalized!) are the only two boolean values. Comparisons like >= and == produce them, and if/while consume them. Under the hood bool is a subtype of int: True behaves as 1 and False as 0.

Truthy and Falsy

Every Python value can stand in for a boolean. The falsy values are: False, None, 0, 0.0, "", [], {}, (), and set(). Everything else is truthy — including " " (a single space), "False" (non-empty text), and [0] (a non-empty list).

NoneType

None is the sole value of type NoneType. It means "no value here": a missing CSV field, a function that returned nothing, a variable initialized before its real value is known. False means "no"; None means "nobody answered."

Syntax Reference

bool() converts any value to True or False using the truthiness rules — a quick way to test what a value would do inside an if.

bool() — convert any value to True/False
print(bool(0))              # Output: False — zero is falsy
print(bool(4500.00))        # Output: True — any non-zero number
print(bool(""))             # Output: False — empty string
print(bool("TXN-4421"))     # Output: True — non-empty string
print(bool([]))             # Output: False — empty list
print(bool([0]))            # Output: True — list with one item (even a zero!)

Comparisons produce booleans directly — you rarely need to write True or False literally; you compute them.

Comparisons — booleans from expressions
credit_score = 720
account_balance = 15420.50

approved = credit_score >= 700
print(approved)                       # Output: True

is_overdrawn = account_balance < 0
print(is_overdrawn)                   # Output: False

Because bool is a subtype of int, True counts as 1 in arithmetic — so sum() over a list of flags counts the True values.

Booleans as numbers — counting flags
fraud_flags = [True, False, True, True, False]

print(sum(fraud_flags))       # Output: 3 — each True adds 1
print(True + True)            # Output: 2 — surprising but consistent

None marks a missing value. It prints as the word None and has its own type, NoneType.

None — representing a missing value
phone = None                  # customer has no phone on file (SQL NULL)
email = "priya@bank.com"

print(phone)                  # Output: None
print(type(phone))            # Output: <class 'NoneType'>

Test for None with is None / is not None. The is operator checks identity (same object), which is exactly right here because there is only one None object in the whole program.

is None — the correct missing-value test
phone = None

if phone is None:
    print("Phone missing — request from customer")
else:
    print(f"Send SMS to {phone}")

# Output: Phone missing — request from customer

Truthiness makes empty-value checks concise: if email: reads as "if email has a value." Use this for strings and collections, but not when 0 or "" are legitimate values you must keep.

Truthiness in if — concise empty checks
discount_code = ""            # customer entered nothing

if discount_code:
    print(f"Applying code {discount_code}")
else:
    print("No discount applied")

# Output: No discount applied

Edge Cases & Gotchas

is compares identity (the same object in memory); == compares value. They can disagree — use == for values, and reserve is for None.

Gotcha — is vs ==
a = [1, 2, 3]
b = [1, 2, 3]

print(a == b)     # Output: True — same contents
print(a is b)     # Output: False — two separate list objects

c = a
print(a is c)     # Output: True — both names point to one object

The subtle distinction that causes real bugs: False, 0, and None are all falsy, but they mean different things. A truthiness check cannot tell them apart — an explicit is None can.

Gotcha — 0 is falsy but not missing
last_txn_amount = 0.0     # a real transaction of zero — NOT missing

if not last_txn_amount:
    print("WRONG: treats a $0.00 transaction as missing data")

if last_txn_amount is None:
    print("This does not print — 0.0 is a value, not None")
else:
    print("Correct: amount present, it just happens to be zero")

# Output:
# WRONG: treats a $0.00 transaction as missing data
# Correct: amount present, it just happens to be zero

Real-World Application

  • ETL — blank CSV fields become None; pipelines test is None before imputing or rejecting rows.
  • Business rules — boolean flags like is_overdraft and computed conditions like credit_score >= 700 gate downstream actions.
  • Analytics — summing boolean columns counts positives; distinguishing 0 from None keeps averages honest.

Key Takeaways

  • True and False are the only booleans; comparisons produce them and conditionals consume them.
  • Falsy values: False, None, 0, 0.0, "", and empty collections. Everything else — including "False" — is truthy.
  • None means "missing," not "no" — always test it with is None, never == None or truthiness.
  • is checks identity; == checks value. Reserve is for None comparisons.
  • bool is an int subtype, so sum() over flags counts the True values.
