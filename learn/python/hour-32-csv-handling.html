CSV Handling

Subtopics: CSV file structure, csv module, reader(), writer(), DictReader, DictWriter

Go to Assignment →

Why CSV Handling Matters in Data Analytics

CSV is the lingua franca of data exchange. Organizations email daily sales CSVs; banks export transaction summaries; Systems export transaction summaries and systems export POS sales — all comma-separated. Python's csv module handles quoting, delimiters, and headers correctly so your ETL scripts don't break on product names containing commas.

Theory & Concepts

  • csv.reader — yields each row as a list of strings.
  • csv.DictReader — yields each row as a dict keyed by column header — preferred for readable ETL code.
  • csv.writer / DictWriter — write rows to CSV files.
  • Always use newline='' when opening files for writing on Windows.

Syntax Reference

csv.reader yields each row as a plain list — you access fields by position. Fine for quick scripts, fragile when column order changes.

csv.reader — rows as lists
import csv

with open("store_sales.csv", "r") as f:
    reader = csv.reader(f)
    header = next(reader)            # first row is the header
    for row in reader:
        print(row)

# Output (example):
# ['101', 'P42', '29.99']
# ['102', 'P17', '49.99']

csv.DictReader yields each row as a dict keyed by the header — code reads by column name, which survives column reordering. This is the preferred interface.

csv.DictReader — rows as dicts
import csv

with open("store_sales.csv", "r") as f:
    reader = csv.DictReader(f)
    total_revenue = 0
    for row in reader:
        total_revenue += float(row["amount"])   # CSV values are strings — cast!

print(f"Total: ${total_revenue:,.2f}")
# Output (example): Total: $79.98

csv.writer writes rows from lists; writerow() takes one row, writerows() takes many.

csv.writer — write rows from lists
import csv

with open("daily_totals.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["store_id", "total"])          # header
    writer.writerows([["S1", 4500], ["S2", 3800]])  # data rows

# daily_totals.csv now contains:
# store_id,total
# S1,4500
# S2,3800

csv.DictWriter writes rows from dicts, with an explicit column order via fieldnames and a one-call header.

csv.DictWriter — write rows from dicts
import csv

fieldnames = ["transaction_id", "account_id", "amount", "txn_type"]
rows = [
    {"transaction_id": "TXN-001", "account_id": "ACC-8842", "amount": "1500.00", "txn_type": "DEBIT"},
    {"transaction_id": "TXN-002", "account_id": "ACC-9103", "amount": "250.00", "txn_type": "CREDIT"},
]

with open("transactions_export.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=fieldnames)
    writer.writeheader()
    writer.writerows(rows)

# transactions_export.csv now has a header line plus two data rows.

Real-World Application

  • Data systems — Export monthly transaction summary as CSV for branch manager review.

Key Takeaways

  • DictReader/DictWriter are the cleanest CSV interface for named columns.
  • CSV values are always strings — cast before arithmetic.
  • Use newline='' when writing CSV on Windows.
