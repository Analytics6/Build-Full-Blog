String Slicing

Subtopics: Slice syntax [start:stop:step], omitting start/stop, reversing strings

Go to Assignment →

Extracting Substrings

Where indexing grabs one character, slicing extracts a whole range: the date out of a timestamp, the domain out of an email, the category code out of a SKU. The syntax is string[start:stop:step], and its single most important rule is that start is inclusive, stop is exclusive — the character at stop is never included.

Theory & Concepts

The Three Slice Parameters

ParameterDefaultMeaning
start0First index included
stopend of stringFirst index excluded
step1Distance between picked characters; negative walks backwards

A helpful consequence of the inclusive/exclusive rule: the slice length is simply stop - start, and adjacent slices like s[:5] and s[5:] fit together with no overlap and no gap.

Why Slicing Never Crashes

Unlike indexing, slicing is forgiving: out-of-bounds values are clipped to the string's ends. "abc"[1:99] quietly returns "bc" instead of raising IndexError. That is convenient, but it also means a typo in a slice bound produces wrong data silently — double-check bounds when parsing fixed-width records.

Reversing with a Negative Step

s[::-1] walks the whole string backwards, producing a reversed copy. With a negative step the defaults flip: start defaults to the end and stop to the beginning.

Syntax Reference

Example — Parsing a pipe-delimited transaction log
# Fixed positions in a transaction log line
txn_log = "2026-07-16T14:32:01|TXN-88421|DEBIT|4500.00"

txn_date = txn_log[0:10]     # chars 0-9 — stop index 10 is excluded
txn_time = txn_log[11:19]    # the HH:MM:SS portion
txn_id = txn_log[20:29]      # the TXN reference

print(txn_date)   # Output: 2026-07-16
print(txn_time)   # Output: 14:32:01
print(txn_id)     # Output: TXN-88421
Example — Omitting start and stop
# Split an email into username and domain
email = "priya.sharma@firstnational.com"
at_index = email.index("@")          # position of the @ separator

username = email[:at_index]           # from the beginning up to @
domain = email[at_index + 1:]         # from after @ to the end

print(username)   # Output: priya.sharma
print(domain)     # Output: firstnational.com
Example — SKU parsing with negative indices
# Retail SKU convention: RET-<category>-<id>-<color>
sku = "RET-ELEC-0042-BLK"

prefix = sku[:3]          # Output slice: RET
category = sku[4:8]       # Output slice: ELEC
variant = sku[-3:]        # last three chars — works for any SKU length

print(f"{prefix} / {category} / {variant}")
# Output: RET / ELEC / BLK

A step of 2 picks every other character; any positive step skips through the string at that interval.

Example — Step values with [::2]
reference = "TXN-8842"

print(reference[::2])     # Output: TN88 — every second character
print(reference[1::2])    # Output: X-42 — every second, starting at index 1

A step of -1 walks the string backwards, producing a reversed copy — the standard Python idiom for reversal.

Example — Reversing with [::-1]
reference = "TXN-8842"

print(reference[::-1])    # Output: 2488-NXT — fully reversed
print("level"[::-1])      # Output: level — palindromes read the same

Unlike indexing, slicing clips out-of-range bounds instead of raising an error. Convenient — but a typo in a bound produces wrong data silently.

Example — Out-of-range slices are clipped
reference = "TXN-8842"

print(reference[4:99])    # Output: 8842 — no IndexError
print(reference[99:])     # Output: (empty string)

Real-World Application

  • ETL — fixed-width mainframe exports are parsed by slicing each field out of the line at known offsets.
  • Data cleaning — timestamp[:10] extracts just the date for daily grouping.
  • Retail analytics — slicing store and category codes out of SKU strings enables regional sales grouping.

Key Takeaways

  • [start:stop:step] — start is inclusive, stop is exclusive, so the slice length is stop - start.
  • Omitted bounds default to the ends of the string: s[:n] and s[n:] partition the string cleanly.
  • Negative indices in slices count from the end; s[-3:] grabs the last three characters of any string.
  • [::-1] reverses; [::2] takes every other character.
  • Slicing clips out-of-bounds values silently — convenient, but check your offsets when parsing fixed-width data.
