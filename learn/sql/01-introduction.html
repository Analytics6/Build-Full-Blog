Introduction to SQL

Understand databases, tables, and rows, then write your first SELECT query.

Every analyst working with relational data eventually asks the same question: "What does the data say?" SQL — Structured Query Language — is how you ask it. Whether you are exploring employee records from a data pipeline or auditing historical rows in an operational database, SQL is the universal interface to relational databases.

Why SQL matters

This course uses a standard sample relational schema with tables such as employees, departments, jobs, and job_history — the same pattern you will see in textbooks and certification exams.

  • Tables and rows — each table stores one kind of entity; each row is one record.
  • Columns — typed attributes such as identifiers, names, dates, and numeric values.
  • Relationships — primary and foreign keys connect related tables for joins and integrity.

Data engineers load tables from files and APIs; analysts query them to build reports and validate pipelines. SQL is the first skill that connects all three roles.

Databases, tables, rows, and columns

A database is an organized collection of data. A table represents one kind of entity — employees, departments, or job assignments. Each row is one record; each column is one attribute.

How SELECT works

The database engine reads the named table, projects the requested columns, and returns a result set your notebook, BI tool, or Python script can consume.

Your first query

SELECT chooses columns; FROM names the table. SQL keywords are conventionally capitalized, though the language is generally case-insensitive.

SQL
-- Browse all employees

SELECT *

FROM hr.employees;



-- Prefer explicit columns in production queries

SELECT employee_id, first_name, last_name, salary

FROM hr.employees;



-- Filter to one department

SELECT employee_id, first_name, last_name, salary

FROM hr.employees

WHERE department_id = 50;

The asterisk (*) means "all columns." Explicit column lists make queries easier to read, prevent accidental exposure of sensitive fields, and reduce data transfer in large pipelines.

sample schema: same language, relational databases

SQL
-- Verify yesterday's load by hire_date

SELECT hire_date, department_id, COUNT(*) AS row_count

FROM hr.employees

GROUP BY hire_date, department_id;



-- Sample recent job history rows

SELECT employee_id, start_date, end_date, job_id, department_id

FROM hr.job_history

LIMIT 10;

You do not need to memorize every table now. The pattern — SELECT … FROM … — is the foundation. Later lessons add filtering, joins, and aggregations on these same tables.

Create a small table locally

Before querying production data, practice on a local SQLite file. This mirrors how developers prototype schemas before deploying to PostgreSQL or Snowflake.

SQL
-- Prototype a mini jobs table

CREATE TABLE hr.jobs (

  job_id     INTEGER PRIMARY KEY,

  job_title  TEXT NOT NULL,

  min_salary REAL,

  max_salary REAL

);



INSERT INTO hr.jobs (job_title, min_salary, max_salary)

VALUES ('Sales Manager', 60000, 85000),

       ('Software Engineer', 45000, 65000);

Real-world context

  • Data engineering — After loading a table from CSV, run a row-count SELECT to confirm the pipeline delivered expected volume.
  • Data analytics — BI dashboards are powered by SELECT queries; Tableau and Metabase generate them automatically, but you must read and debug them.
  • Reporting — Aggregate analysis often starts with SQL: pull grouped totals from joined tables before building dashboards.

Practice

  1. Write a query that returns only job_title and max_salary from the jobs table.
  2. List three columns you would expect in a job_history table and explain why each matters.
  3. Open SQLite, create a two-row employees table, and SELECT all rows.

Key takeaways

  • Relational databases organize records into tables of rows and columns.
  • SELECT … FROM … is the foundation of every read query on relational tables.
  • Data engineers, analysts, and scientists all rely on SQL to inspect, validate, and transform relational data.
  • SQLite provides a zero-setup environment for practicing before querying production systems.
