Input Function

Subtopics: input() function, reading strings, type conversion of input (int(), float())

Go to Assignment →

Talking to Your Program

So far your programs have only produced output. The input() function adds the other direction: it pauses the program, shows a prompt, and waits for the user to type something and press Enter. The single most important fact about it — the one behind countless beginner bugs — is that input() always returns a string, no matter what the user typed.

Theory & Concepts

  • input(prompt) prints the prompt (no automatic newline), blocks until Enter, and returns the typed text as a str.
  • To do arithmetic, cast with int() or float() — an invalid cast raises ValueError.
  • Chain .strip() immediately to remove stray spaces users habitually type.
  • Production pipelines rarely use input(), but ad-hoc CLIs, admin tools, and training simulators rely on it heavily.

Syntax Reference

The basic call: prompt, wait, receive a string.

input() — read text from the user
customer_id = input("Enter customer ID: ")   # user types: C-20481

print(f"Looking up {customer_id}...")
# Output: Looking up C-20481...

print(type(customer_id))
# Output: <class 'str'> — ALWAYS a string

Numeric input must be cast before math — otherwise + concatenates and * repeats, silently producing nonsense.

int() around input() — numeric input
birth_year = int(input("Birth year: "))   # user types: 1998

age = 2026 - birth_year
print(f"Age: {age}")   # Output: Age: 28

# Without int(): "1998" - 2026 would raise TypeError,
# and "1998" * 2 would print 19981998.

Use float() for amounts with decimals — the standard pattern for money and rates.

float() around input() — decimal amounts
income = float(input("Annual income: "))          # user types: 850000
loan_amount = float(input("Requested loan: "))     # user types: 300000

ratio = loan_amount / income
print(f"Debt-to-income ratio: {ratio:.2f}")
# Output: Debt-to-income ratio: 0.35

Chain string methods directly onto input() to clean the value in the same line — trimming whitespace and normalizing case.

input() with .strip().title() — clean as you read
name = input("Enter your name: ").strip().title()   # user types: "  priya sharma "

print(f"Welcome, {name}!")
# Output: Welcome, Priya Sharma!

Edge Cases & Gotchas

If the user types something that is not a number, the cast raises ValueError and the program crashes. Guard with .isdigit() for whole numbers (full error handling arrives in Hour 30).

Gotcha — invalid input crashes the cast
raw = input("Quantity: ")          # user types: twelve

# quantity = int(raw)             <-- ValueError: invalid literal for int()

if raw.isdigit():                  # pre-check before casting
    quantity = int(raw)
    print(f"Ordering {quantity} units")
else:
    print("Please enter a whole number")

# Output: Please enter a whole number

Real-World Application

  • Admin CLIs — scripts that let operations staff look up an account or trigger a pipeline rerun by typing an ID.
  • Calculators — quick eligibility and pricing tools: read two numbers, print a ratio or total.
  • Prototyping — before building a web form, an input() loop tests the underlying logic interactively.

Notebook & Practice

  • Ask for a name and greet the user with it, cleaned by .strip().title().
  • Read a birth year with int(input(...)) and compute the age.
  • Build a two-input debt-to-income calculator with float().
  • Type letters into a numeric prompt, read the ValueError, then add an .isdigit() guard.

Key Takeaways

  • input() always returns a string — cast with int()/float() before any math.
  • Invalid casts raise ValueError; guard with .isdigit() or (later) try/except.
  • Chain .strip() onto input() to clean whitespace immediately.
  • Interactive scripts complement scheduled pipelines for ad-hoc operations work.
