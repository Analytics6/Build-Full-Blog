Methods in Class

Subtopics: Instance methods, accessing instance attributes inside methods, modifying attribute values via methods, returning values from methods

Go to Assignment →

Why Class Methods Matter in Data Analytics

Attributes store what an object is; methods define what it does. A BankAccount doesn't just hold a balance — it can deposit(), withdraw(), and get_statement(). Methods bundle behavior with data, so domain logic stays organized: call account.deposit() instead of scattering updates across procedural scripts.

Theory & Concepts

Instance Methods

Functions defined inside a class. The first parameter is always self, giving access to the object's attributes and other methods.

Reading vs Mutating State

  • Getter-style — return computed values without changing state: get_tax()
  • Mutator-style — modify attributes: deposit(amount) updates self.balance

Method Calls

Call via the instance: account.deposit(500). Python passes self automatically.

Syntax Reference

A mutator method changes the object's state — here, deposit() updates self.balance and returns the new value.

Mutator method — deposit() changes state
class BankAccount:
    def __init__(self, account_id, balance=0.0):
        self.account_id = account_id
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount        # mutate the instance
        return self.balance

acct = BankAccount("ACC-8842", 1000.0)
acct.deposit(250.0)

print(acct.balance)    # Output: 1250.0

Methods can enforce business rules — withdraw() refuses to overdraw the account, keeping the rule in exactly one place.

Guarded method — withdraw() with a rule
class BankAccount:
    def __init__(self, account_id, balance=0.0):
        self.account_id = account_id
        self.balance = balance

    def withdraw(self, amount):
        if amount > self.balance:
            return "DECLINED — insufficient funds"
        self.balance -= amount
        return self.balance

acct = BankAccount("ACC-8842", 500.0)

print(acct.withdraw(200.0))    # Output: 300.0
print(acct.withdraw(900.0))    # Output: DECLINED — insufficient funds

A getter-style method computes and returns a value without touching state — get_tax() derives tax from the balance each time it is called.

Getter method — get_tax() computes a value
class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.salary = salary

    def get_tax(self):
        return round(self.salary * 0.10, 2)      # reads, never writes

emp = Employee("Alice", 75000)

print(emp.get_tax())    # Output: 7500.0
print(emp.salary)       # Output: 75000 — unchanged

Methods can call other methods on the same object through self — composing small behaviors into summaries.

Methods calling methods — self.get_tax()
class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.salary = salary

    def get_tax(self):
        return round(self.salary * 0.10, 2)

    def profile(self):
        return f"{self.name} — salary ${self.salary}, tax ${self.get_tax()}"

emp = Employee("Alice", 75000)
print(emp.profile())
# Output: Alice — salary $75000, tax $7500.0

Real-World Application

  • User models — account.verify_pin(), account.upgrade_tier() in a web portal.
  • Inventory data — product.reduce_stock(qty) enforces business rules in one place.
  • Data validation — transaction.is_suspicious() encapsulates fraud-check logic.

Key Takeaways

  • Methods are functions bound to objects; self is always the first parameter.
  • Methods can read state, compute values, or mutate attributes.
  • Encapsulating behavior in methods keeps data analysis domain logic maintainable.
  • Call methods on instances: obj.method() — Python handles self.
