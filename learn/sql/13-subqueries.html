Subqueries

Nest SELECT inside another query — filter, compute, and compare using inline result sets.

Go to Assignment →

A subquery is a SELECT nested inside another statement. Use it when you need an intermediate result — the average transaction salary, the list of qualifying IDs, or the groups exceeding an aggregate threshold — before the outer query can finish.

Subquery execution flow

Subquery in WHERE with IN

SQL
-- Sales to premium hr.employees
SELECT employee_id, employee_id, salary
FROM hr.employees
WHERE employee_id IN (
  SELECT employee_id
  FROM hr.employees
  WHERE department_id = '50'
);

-- hr.jobs never sold (anti-pattern via NOT IN)
SELECT job_title
FROM hr.jobs
WHERE job_id NOT IN (
  SELECT DISTINCT job_id FROM hr.job_history
);

Scalar subquery — single value

Returns one row, one column. Use in SELECT list or WHERE comparisons.

SQL
-- Sales above the overall average
SELECT employee_id, salary
FROM hr.employees
WHERE salary > (
  SELECT AVG(salary) FROM hr.employees
);

-- Include average in each row for comparison
SELECT employee_id, salary,
       salary - (SELECT AVG(salary) FROM hr.job_history) AS diff_from_avg
FROM hr.job_history;

Subquery in FROM (derived table)

SQL
-- Revenue per store, then filter high performers
SELECT department_id, revenue
FROM (
  SELECT department_id, SUM(salary) AS revenue
  FROM hr.employees
  GROUP BY department_id
) AS store_totals
WHERE revenue > 100000;

Derived tables in FROM require an alias. CTEs (Lesson 26) offer cleaner syntax for the same pattern.

EXISTS — test for matching rows

SQL
-- hr.employees who have at least one loan application
SELECT c.employee_id, c.last_name
FROM hr.employees AS c
WHERE EXISTS (
  SELECT 1
  FROM hr.job_history AS la
  WHERE la.employee_id = c.employee_id
);

EXISTS stops at the first match — often faster than IN on large subquery results.

Real-world context

  • Threshold filtering — Find the jobs table priced above category average for markdown analysis.
  • Cohort analysis — Subquery identifies the employees table with > 5 the job_history table, outer query analyzes their behavior.
  • Data validation — Find job_history rows whose employee_id is NOT IN the employees table (orphan check).

Practice

  1. Find the job_history table with salary greater than the average deposit salary.
  2. List the jobs table in categories that have more than 20 items (subquery + IN).
  3. Rewrite an IN subquery using EXISTS for the employees table with sales.

Key takeaways

  • Subqueries nest SELECT inside WHERE, FROM, or SELECT clauses.
  • Scalar subqueries return a single value; IN/EXISTS test membership.
  • Derived tables in FROM require aliases and enable filtering on aggregates.
  • Prefer EXISTS over IN for large result sets; use JOINs when readability matters.
