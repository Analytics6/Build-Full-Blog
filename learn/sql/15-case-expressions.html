CASE Expressions

Conditional logic inside SELECT — bucket the employees table, classify the job_history table, and build derived columns for analytics.

Go to Assignment →

Not every answer is a raw column value. Analysts need buckets — "high / medium / low spender," "approved / rejected / pending" — and data scientists need encoded categories for models. CASE expressions bring if/else logic directly into SQL, avoiding round-trips to Python for simple transformations.

CASE evaluation flow

Searched CASE — flexible conditions

SQL
-- Bucket hr.employees by lifetime spend
SELECT employee_id, last_name,
  CASE
    WHEN lifetime_spend >= 5000 THEN 'Platinum'
    WHEN lifetime_spend >= 1000 THEN 'Gold'
    WHEN lifetime_spend >= 100  THEN 'Silver'
    ELSE 'Bronze'
  END AS tier
FROM (
  SELECT c.employee_id, c.last_name,
         COALESCE(SUM(s.salary), 0) AS lifetime_spend
  FROM hr.employees c
  LEFT JOIN hr.employees s ON s.employee_id = c.employee_id
  GROUP BY c.employee_id, c.last_name
);

Simple CASE — match a single expression

SQL
-- Friendly labels for transaction types
SELECT employee_id, salary,
  CASE job_id
    WHEN 'deposit'    THEN 'Money In'
    WHEN 'transfer' THEN 'Money Out'
    WHEN 'transfer'   THEN 'Transfer'
    ELSE 'Other'
  END AS txn_label
FROM hr.job_history;

CASE in aggregations — pivot-like counts

SQL
-- Count loan outcomes by status in one row per month
SELECT
  strftime('%Y-%m', submitted_at) AS month,
  SUM(CASE WHEN status = 'approved' THEN 1 ELSE 0 END) AS approved_count,
  SUM(CASE WHEN status = 'rejected' THEN 1 ELSE 0 END) AS rejected_count,
  SUM(CASE WHEN status = 'pending'   THEN 1 ELSE 0 END) AS pending_count
FROM hr.job_history
GROUP BY month;

CASE with ORDER BY — custom sort

SQL
-- Sort hr.jobs: Sales first, then alphabetical
SELECT job_title, category, salary
FROM hr.jobs
ORDER BY
  CASE category
    WHEN 'Sales' THEN 1
    WHEN 'IT'  THEN 2
    ELSE 3
  END,
  job_title;

Real-world context

  • Feature engineering — CASE buckets raw salary into risk tiers for ML models on the job_history table.
  • Executive reporting — Conditional aggregation produces approved/rejected/pending counts without three separate queries.
  • Data quality flags — CASE marks rows as 'valid', 'missing_price', or 'negative_qty' during ETL validation.

Practice

  1. Classify the employees table as 'Large' (>200), 'Medium' (50–200), or 'Small' (<50) by salary.
  2. Count employee records and transfers per account using CASE inside SUM.
  3. Add a CASE column labeling the jobs table as 'Budget' or '50' based on salary threshold.

Key takeaways

  • CASE adds conditional logic to SELECT, WHERE, ORDER BY, and aggregate expressions.
  • Searched CASE (WHEN condition) is more flexible; simple CASE matches one expression.
  • Conditional aggregation with CASE replaces multiple queries for pivot-style reports.
  • Always include ELSE to avoid unexpected NULLs in derived columns.
