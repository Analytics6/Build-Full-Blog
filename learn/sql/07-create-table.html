CREATE TABLE and Data Types

Define schemas and choose column types — the foundation every database developer lays before loading HR datasets.

Go to Assignment →

Before a single row lands in the employees table or the job_history table, someone defines the schema: column names, data types, and constraints. CREATE TABLE is DDL (Data Definition Language) — it shapes the structure that all downstream SELECT and INSERT statements depend on.

Create the job title catalog

SQL
CREATE TABLE hr.jobs (
  job_id   INTEGER PRIMARY KEY,
  job_title TEXT NOT NULL,
  min_salary   REAL,
  salary   DECIMAL(10, 2) NOT NULL,
  job_id          TEXT,
  created_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Create related tables

SQL
CREATE TABLE hr.employees (
  employee_id      INTEGER PRIMARY KEY,
  hire_date    DATE NOT NULL,
  department_id     INTEGER NOT NULL,
  employee_id  INTEGER,
  salary DECIMAL(12, 2),
  status       TEXT DEFAULT 'completed'
);

CREATE TABLE hr.job_history (

);

);

Choosing the right type

  • Use DECIMAL for money — never REAL/FLOAT (floating-point rounding errors).
  • Use INTEGER for IDs and counts; TEXT for open-ended strings.
  • Pick TIMESTAMP when time-of-day matters; DATE for calendar dates only.
  • NOT NULL on required fields prevents silent data-quality gaps in ETL.

Real-world context

  • ETL — Schema definitions live in migration scripts version-controlled alongside Python ETL code.
  • Warehouse design — Snowflake and BigQuery map these types to their own equivalents during ingestion.
  • Analytics — Correct types ensure pandas reads amounts as numeric, not strings, when exported via SQL.

Practice

  1. Create a the job_history table with id, employee_id, requested_amount, salary, status, and submitted_at.
  2. Explain why salary should be DECIMAL, not REAL.
  3. Add a DEFAULT value for status on new loan applications.

Key takeaways

  • CREATE TABLE defines column names, types, and inline constraints.
  • Choose types that match the data and prevent precision loss (DECIMAL for money).
  • NOT NULL and DEFAULT enforce data quality at insert time.
  • Schema design precedes every ETL pipeline loading HR data.
