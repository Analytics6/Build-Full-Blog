Lambda Functions

Subtopics: Lambda syntax, anonymous functions, map() basics, filter() basics, sorting with key=lambda

Go to Assignment →

Tiny Functions, Inline

Many transformations are one-liners: add tax to a price, test whether an amount exceeds a threshold, pick the field to sort by. Defining a full def for each clutters the code. A lambda is an anonymous, single-expression function written right where it is used — most often as the function argument to map(), filter(), and sorted(), and later in Pandas .apply().

Theory & Concepts

Syntax

lambda arguments: expression — no name, no return keyword (the expression's value is returned automatically), and strictly one expression: no statements, no multi-line logic.

When to Use — and Not

  • Use when passing a short throwaway function to another function.
  • Avoid when the logic needs a name, a docstring, or more than one expression — write a def.
  • Assigning a lambda to a variable (f = lambda x: ...) works but defeats the purpose; PEP 8 recommends def there.

Syntax Reference

The anatomy: parameters before the colon, a single returned expression after it.

lambda — anatomy of an anonymous function
add_three = lambda a, b, c: a + b + c     # for demonstration only

print(add_three(10, 20, 30))    # Output: 60

# Equivalent def version:
def add_three_def(a, b, c):
    return a + b + c

With map(), a lambda transforms every element of a list — here, adding 18% tax to each price.

map() with lambda — transform every price
prices = [19.99, 45.00, 8.50, 120.00]

with_tax = list(map(lambda p: round(p * 1.18, 2), prices))

print(with_tax)
# Output: [23.59, 53.1, 10.03, 141.6]

With filter(), the lambda is a yes/no test — only elements where it returns a truthy value survive.

filter() with lambda — keep high-value transactions
transactions = [
    {"id": "T1", "amount": 500},
    {"id": "T2", "amount": 15000},
    {"id": "T3", "amount": 200},
]

high_value = list(filter(lambda t: t["amount"] > 10000, transactions))

print(high_value)
# Output: [{'id': 'T2', 'amount': 15000}]

With sorted(key=...), the lambda picks what to sort by — here, ordering customer dicts by spend, highest first.

sorted() with key=lambda — order by a field
customers = [
    {"name": "Priya", "spend": 8800},
    {"name": "James", "spend": 7200},
    {"name": "Maria", "spend": 9500},
]

top_first = sorted(customers, key=lambda c: c["spend"], reverse=True)

print(top_first[0]["name"])    # Output: Maria — highest spend
print([c["name"] for c in top_first])
# Output: ['Maria', 'Priya', 'James']

A lambda with a conditional expression handles simple either/or transforms inline.

lambda with a conditional — inline branching
amounts = [500, 15000, 200, 82000]

labels = list(map(lambda a: "HIGH" if a > 10000 else "normal", amounts))

print(labels)
# Output: ['normal', 'HIGH', 'normal', 'HIGH']

Edge Cases & Gotchas

Comprehensions often express the same transform more readably than map/filter with lambdas — know both, prefer whichever reads cleaner in context.

Lambda vs comprehension — same result, two styles
prices = [19.99, 45.00, 8.50]

# map + lambda
taxed_1 = list(map(lambda p: round(p * 1.18, 2), prices))

# list comprehension — many teams find this clearer
taxed_2 = [round(p * 1.18, 2) for p in prices]

print(taxed_1)   # Output: [23.59, 53.1, 10.03]
print(taxed_2)   # Output: [23.59, 53.1, 10.03]

Real-World Application

  • Pandas — df["email"].apply(lambda x: x.strip().lower()) normalizes a column before deduplication.
  • Ranking — sorted(items, key=lambda x: x[2]) orders tuples by any field for top-N reports.
  • Filtering — one-line threshold filters select approved or high-value records in memory.

Key Takeaways

  • lambda args: expression — anonymous, single-expression, auto-returning.
  • map() transforms, filter() selects, sorted(key=) orders by a computed field.
  • Reach for def once logic exceeds one expression or deserves a name.
  • Comprehensions often read better than map/filter — choose per case.
  • Lambdas are everywhere in Pandas .apply() — this hour pays off in the Pandas week.
