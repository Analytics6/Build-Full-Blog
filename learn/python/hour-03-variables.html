Variables

Subtopics: Variable creation, naming rules, dynamic typing, re-assignment

Go to Assignment →

What a Variable Really Is

A variable in Python is a name that points to an object in memory — a label, not a box. When you write price = 49.99, Python creates a float object and binds the name price to it. This mental model matters: assignment never copies data, it only attaches another label. Two names can point to the very same object, which becomes important when you work with lists and dictionaries later in the course.

Theory & Concepts

Creating Variables

Python creates a variable the moment you assign a value with =. There is no declaration keyword and no type annotation required — this is dynamic typing. The type lives on the object, not on the name.

Naming Rules

  • Must start with a letter (a–z, A–Z) or an underscore _.
  • Cannot start with a digit — 2price = 10 raises SyntaxError.
  • Case-sensitive: total_sales and Total_Sales are two different variables.
  • Reserved keywords (if, for, class, return, ...) cannot be used as names.
  • Convention: use snake_case for variables — transaction_amount, customer_id.

Common pitfall: Python will happily let you overwrite built-in names like list, sum, or type. Writing sum = 100 works — until three cells later sum([1, 2, 3]) fails with TypeError: 'int' object is not callable. Avoid built-in names entirely.

Re-assignment & Multiple Assignment

A name can be rebound to a new object at any time — even one of a different type. Python also supports assigning several names at once through tuple unpacking, which enables the famous one-line swap.

Syntax Reference

Example — Product inventory variables
# Describe one product from a retail catalog
product_name = "Wireless Keyboard"   # str
unit_price = 49.99                    # float
quantity_in_stock = 120               # int
is_discontinued = False               # bool

# Derive a new value from existing variables
total_value = unit_price * quantity_in_stock

print(f"{product_name}: ${total_value:,.2f} in stock")
# Output: Wireless Keyboard: $5,998.80 in stock
Example — Dynamic typing in action
# The same name can point to different types over time
account_balance = 15000.50
print(type(account_balance))   # Output: <class 'float'>

account_balance = "FROZEN"     # legal — the name now points to a string
print(type(account_balance))   # Output: <class 'str'>

# Flexibility cuts both ways: downstream code expecting a number
# will now crash. Keep a variable's type stable in real projects.
Example — Multiple assignment in one line
# Assign three related values in one statement
debit, credit, balance = 500, 1200, 8700

print(debit)      # Output: 500
print(credit)     # Output: 1200
print(balance)    # Output: 8700

The same unpacking mechanism enables Python's famous one-line swap — no temporary variable required.

Example — Swapping two variables
checking = 3200
savings = 9800

checking, savings = savings, checking   # swap in one line

print(checking)   # Output: 9800
print(savings)    # Output: 3200
Example — Bank customer snapshot
# Combine several variables into one formatted report line
customer_id = "CUS-20481"
loyalty_tier = "gold"
account_balance = 15420.50
txn_count_30d = 12                # transactions in the last 30 days

print(f"{customer_id}: {loyalty_tier} tier, "
      f"balance ${account_balance:,.2f}, {txn_count_30d} recent txns")
# Output: CUS-20481: gold tier, balance $15,420.50, 12 recent txns

Real-World Application

  • ETL scripts — variables hold file paths, row counts, and connection strings: source_path = "/data/bank_transactions.csv".
  • Notebook analysis — intermediate metrics like avg_balance and txn_count_30d get named so later cells can reuse them.
  • Configuration — constants by convention use UPPER_CASE: MAX_RETRIES = 3, TAX_RATE = 0.18. Python doesn't enforce it, but every reader understands the intent.

Notebook & Practice

  • Declare variables for a product's name, price, and quantity; compute and print the total cost.
  • Swap two variables in one line with a, b = b, a and verify the result.
  • Try 2price = 10 and read the SyntaxError message carefully.
  • Reassign an int variable to a string; print type() before and after.
  • Assign sum = 5, then try sum([1, 2, 3]) — observe the error, then restart the kernel.

Key Takeaways

  • Variables are name-to-object bindings created by =; assignment attaches a label, it does not copy data.
  • Use snake_case; names cannot start with digits or shadow keywords — and avoid shadowing built-ins like sum and list.
  • Dynamic typing lets a name point to a new type at any time; use that power sparingly to keep code predictable.
  • Tuple unpacking assigns multiple names at once: x, y = 1, 2, and a, b = b, a swaps in place.
