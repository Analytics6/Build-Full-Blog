Working with Numbers

Nearly every program computes something: totals, averages, percentages, remainders. Python gives you two numeric workhorses — integers for whole numbers and floats for decimals — plus seven arithmetic operators and a family of built-in functions. This hour walks through each of them individually, with a runnable example for every one.

Theory & Concepts

Integer (int)

Whole numbers with arbitrary precision — a Python int can grow as large as memory allows, so 2 ** 1000 just works. Use ints for counts, IDs, and quantities.

Float (float)

Decimal numbers stored as 64-bit binary doubles. Because binary cannot represent every decimal fraction exactly, floats carry tiny rounding errors: 0.1 + 0.2 is not exactly 0.3. For analytics this is fine — compare with a tolerance or round for display. Production financial systems use the decimal module for exact arithmetic.

Operator Precedence

Highest to lowest: parentheses → ** → * / // % → + -. When in doubt, add parentheses — clarity beats memorized precedence tables.

Arithmetic Operators

Addition, subtraction, and multiplication behave exactly as you expect from arithmetic. They keep ints as ints and produce floats as soon as one operand is a float.

Operators + - * — basic arithmetic
# Daily revenue math for one store
morning_sales = 1250
evening_sales = 980

total_sales = morning_sales + evening_sales
print(total_sales)              # Output: 2230

print(total_sales - 230)        # Output: 2000
print(total_sales * 2)          # Output: 4460

True division / always returns a float, even when the result is a whole number. This surprises many beginners: 10 / 5 is 2.0, not 2.

Operator / — division always returns float
# Average transaction value
total_amount = 4500
txn_count = 4

print(total_amount / txn_count)   # Output: 1125.0
print(10 / 5)                     # Output: 2.0 — float even when exact

Floor division // divides and rounds down to the nearest whole number. Careful with negatives: it floors toward negative infinity, so -7 // 2 is -4, not -3.

Operator // — floor division
# How many full batches of 500 fit into 4230 transactions?
transactions = 4230
batch_size = 500

print(transactions // batch_size)   # Output: 8
print(-7 // 2)                      # Output: -4 — floors toward negative infinity

Modulo % returns the remainder after division. Paired with // it answers "how many full groups, and how much is left over?"

Operator % — modulo (remainder)
# Rows left over after packing into batches of 500
transactions = 4230
batch_size = 500

print(transactions % batch_size)   # Output: 230
print(17 % 5)                      # Output: 2
print(10 % 2)                      # Output: 0 — even numbers have remainder 0

Exponentiation ** raises a number to a power. It binds tighter than multiplication, so 3 * 2 ** 2 is 12, not 36.

Operator ** — exponentiation
# Compound interest: 5% per year for 3 years
principal = 10000.00
growth_factor = (1 + 0.05) ** 3

print(round(growth_factor, 4))            # Output: 1.1576
print(round(principal * growth_factor, 2)) # Output: 11576.25

Numeric Built-in Functions

int() converts a value to an integer. From a float it truncates toward zero (it never rounds); from a string it requires plain integer text.

int() — convert to integer
print(int(49.99))     # Output: 49 — truncated, not rounded
print(int(-49.99))    # Output: -49 — truncates toward zero
print(int("120"))     # Output: 120 — string of digits is fine

# int("49.99")        <-- ValueError: go through float() first
print(int(float("49.99")))   # Output: 49

float() converts a value to a decimal number — the standard first step when a CSV column of amounts arrives as text.

float() — convert to decimal
raw_amount = "1549.99"          # str straight from a store_sales CSV

sale_amount = float(raw_amount)
print(sale_amount)               # Output: 1549.99
print(float(12))                 # Output: 12.0
print(sale_amount * 2)           # Output: 3099.98 — now math works

abs() returns the absolute value — the distance from zero. Use it to measure the size of a change regardless of direction, such as a debit vs a credit.

abs() — absolute value
# A negative amount means money left the account
balance_change = -750.25

print(abs(balance_change))   # Output: 750.25
print(abs(300))              # Output: 300 — positives pass through unchanged

round() rounds to a given number of decimal places (default 0). Note that Python uses banker's rounding on exact halves — it rounds to the nearest even number.

round() — round to a precision
avg_basket = 47.8567

print(round(avg_basket, 2))   # Output: 47.86
print(round(avg_basket))      # Output: 48 — no digits means whole number

# Banker's rounding: exact halves go to the nearest EVEN number
print(round(2.5))             # Output: 2
print(round(3.5))             # Output: 4

pow() is the function form of **. Its three-argument form pow(base, exp, mod) computes (base ** exp) % mod efficiently — used in cryptography.

pow() — power function
print(pow(2, 10))       # Output: 1024 — same as 2 ** 10
print(pow(9, 0.5))      # Output: 3.0 — fractional exponent = square root
print(pow(7, 2, 5))     # Output: 4 — (7**2) % 5, computed efficiently

divmod() returns the quotient and the remainder in one call — a tuple of (a // b, a % b). Perfect for splitting totals into units.

divmod() — quotient and remainder together
# Convert 4230 seconds of batch runtime into minutes and seconds
minutes, seconds = divmod(4230, 60)

print(minutes)    # Output: 70
print(seconds)    # Output: 30
print(divmod(17, 5))   # Output: (3, 2)

min() and max() return the smallest and largest of their arguments — or of a single iterable like a list.

min() / max() — smallest and largest
daily_sales = [1250, 980, 1450, 875, 1620]

print(min(daily_sales))       # Output: 875 — worst day
print(max(daily_sales))       # Output: 1620 — best day
print(max(300, 120, 950))     # Output: 950 — works on plain arguments too

sum() adds up every number in an iterable. Its optional second argument is a starting value.

sum() — total of a sequence
daily_sales = [1250, 980, 1450, 875, 1620]

print(sum(daily_sales))          # Output: 6175
print(sum(daily_sales) / len(daily_sales))   # Output: 1235.0 — the average
print(sum(daily_sales, 1000))    # Output: 7175 — starts counting from 1000

The math module adds scientific functions beyond the built-ins: square roots, ceilings, floors, and constants like pi.

math module — sqrt, ceil, floor, pi
import math

print(math.sqrt(144))      # Output: 12.0
print(math.ceil(4.01))     # Output: 5 — always rounds UP
print(math.floor(4.99))    # Output: 4 — always rounds DOWN
print(round(math.pi, 4))   # Output: 3.1416

Edge Cases & Gotchas

Floating-point rounding is the classic numeric trap. Never compare floats for exact equality — round first or use a tolerance.

Gotcha — floating-point precision
print(0.1 + 0.2)              # Output: 0.30000000000000004
print(0.1 + 0.2 == 0.3)       # Output: False!

# Fix 1: round before comparing
print(round(0.1 + 0.2, 2) == 0.3)      # Output: True

# Fix 2: compare with a tolerance
import math
print(math.isclose(0.1 + 0.2, 0.3))    # Output: True

Real-World Application

  • Inventory — divmod() splits a stock count into full pallets plus leftover units in one call.
  • Banking — ** models compound interest; math.isclose() prevents one-cent reconciliation bugs.
  • Analytics — sum() / len() is the simplest average; min()/max() find best and worst performing days.

Key Takeaways

  • / always returns a float; // floors; % gives the remainder; divmod() returns both at once.
  • int() truncates toward zero — use round() when you want rounding.
  • min(), max(), and sum() work directly on lists — the foundation of quick analytics.
  • The math module adds sqrt, ceil, floor, and pi.
  • Never compare floats with ==; use round() or math.isclose().
