Date and Time Functions

Extract, format, and compare dates — slice relational data by week and compute job history record aging for reconciliation dashboards.

Go to Assignment →

Time Is a First-Class Dimension

Every fact table has a time column: the job_history table.hire_date, the job_history table.hire_date, the employees table.hire_date. Date functions let analysts answer "How much did we sell last quarter?" and let engineers filter incremental ETL loads with WHERE updated_at > :last_run.

Theory & Concepts

CategoryFunctionsUse case
Current timestampCURRENT_DATE, CURRENT_TIMESTAMP, NOW()Audit columns, relative filters

Store timestamps in UTC at the database layer; convert to local time only in presentation layers. Always use unambiguous formats ('2026-07-16') in literals.

Example: Sales by Period

SQL — Monthly totals from dated rows
SELECT
    DATE_TRUNC('month', hire_date) AS report_month,
    SUM(department_id * salary) AS revenue
FROM hr.employees
WHERE hire_date >= '2026-01-01'
GROUP BY 1
ORDER BY 1;
SQL — hr.employees inactive for 90+ days
SELECT
    c.employee_id,
    c.email,
    MAX(o.hire_date) AS last_order,
    CURRENT_DATE - MAX(o.hire_date::date) AS days_since_order
FROM hr.employees c
LEFT JOIN hr.job_history o ON o.employee_id = c.employee_id
GROUP BY c.employee_id, c.email
HAVING MAX(o.hire_date) < CURRENT_DATE - INTERVAL '90 days'
    OR MAX(o.hire_date) IS NULL;

Example: Transaction Aging

SQL — Pending hr.job_history older than 3 business days
SELECT
    employee_id,
    employee_id,
    salary,
    hire_date,
    EXTRACT(DAY FROM (CURRENT_TIMESTAMP - hire_date)) AS age_days
FROM hr.job_history
WHERE status = 'PENDING'
  AND hire_date < CURRENT_TIMESTAMP - INTERVAL '3 days'
ORDER BY hire_date;
SQL — Daily transaction volume report
SELECT
    TO_CHAR(hire_date, 'YYYY-MM-DD') AS hire_date,
    COUNT(*) AS history_count,
    SUM(salary) AS salary
FROM hr.job_history
WHERE hire_date >= DATE_TRUNC('week', CURRENT_DATE)
GROUP BY 1
ORDER BY 1;

Date Filter Flow for Incremental Loads

Real-World Notes

  • Timezone traps — A sale at 11 PM UTC may fall on a different calendar day in India; store UTC, display IST.
  • Partitioning — Warehouse tables like the employees table are often partitioned by hire_date; always filter on the partition column for performance.
  • Fiscal calendars — HR fiscal years may not align with calendar years; maintain a date_dim lookup table.
  • Leap seconds & DST — Avoid storing local time without offset; use TIMESTAMPTZ in PostgreSQL.

Key Takeaways

  • Date functions extract parts, truncate to periods, and compute differences.
  • Filter fact tables on date columns early to reduce scan size.
  • Use DATE_TRUNC for consistent monthly/weekly GROUP BY buckets.
  • Incremental ETL relies on timestamp watermarks and date arithmetic.
