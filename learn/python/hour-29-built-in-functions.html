Built-in Functions

Subtopics: len(), type(), isinstance(), range(), enumerate(), zip(), map(), filter(), sorted(), reversed(), any(), all(), input(), print() options

Always Available, Always Useful

Python ships with several dozen functions that need no import — they are simply there, in every script and notebook. Before reaching for Pandas or NumPy, these built-ins handle the everyday work: counting, inspecting, pairing, transforming, filtering, and validating. This hour covers each of the important ones individually.

Inspection

len() counts the elements of any sequence or collection — characters in a string, items in a list, pairs in a dict.

len() — count elements
transactions = [1500.00, 250.75, 4500.00]

print(len(transactions))       # Output: 3
print(len("ACC-8842"))         # Output: 8 — characters in a string
print(len({"a": 1, "b": 2}))   # Output: 2 — key-value pairs

type() reveals the exact class of any value — the first thing to check when a calculation misbehaves.

type() — inspect a value's class
amount = "4500.00"     # looks numeric, but is it?

print(type(amount))         # Output: <class 'str'> — aha, needs casting
print(type(float(amount)))  # Output: <class 'float'>

isinstance(value, type) tests whether a value is of a given type (or a subclass). Prefer it over type(x) == ... in real code — it reads better and respects inheritance.

isinstance() — type checking done right
amount = 4500.00

print(isinstance(amount, float))          # Output: True
print(isinstance(amount, str))            # Output: False
print(isinstance(amount, (int, float)))   # Output: True — accepts a tuple of types

Iteration Helpers

range(start, stop, step) generates a sequence of integers — the engine behind counted loops. The stop value is excluded, matching slice behavior.

range() — generate number sequences
print(list(range(5)))          # Output: [0, 1, 2, 3, 4]
print(list(range(1, 6)))       # Output: [1, 2, 3, 4, 5]
print(list(range(0, 20, 5)))   # Output: [0, 5, 10, 15] — step of 5

enumerate(iterable, start) yields (index, value) pairs as you loop — no manual counter variable needed. Use start=1 for human-friendly numbering.

enumerate() — loop with an index
categories = ["Electronics", "Clothing", "Groceries"]

for num, category in enumerate(categories, start=1):
    print(f"{num}. {category}")

# Output:
# 1. Electronics
# 2. Clothing
# 3. Groceries

zip(a, b) pairs elements from parallel iterables, stopping at the shortest. Feeding it to dict() builds a lookup table in one line.

zip() — pair parallel lists
store_ids = ["S001", "S002", "S003"]
revenues = [42000, 38500, 51200]

for store, revenue in zip(store_ids, revenues):
    print(f"{store}: ${revenue:,}")
# Output:
# S001: $42,000
# S002: $38,500
# S003: $51,200

store_revenue = dict(zip(store_ids, revenues))
print(store_revenue["S002"])    # Output: 38500

Transforming & Filtering

map(function, iterable) applies a function to every element, returning a lazy iterator — wrap in list() to see the results.

map() — apply a function to every element
raw_amounts = ["1500.00", "250.75", "4500.00"]   # strings from a CSV

amounts = list(map(float, raw_amounts))

print(amounts)        # Output: [1500.0, 250.75, 4500.0]
print(sum(amounts))   # Output: 6250.75 — now math works

filter(function, iterable) keeps only the elements for which the function returns a truthy value — also lazy, also wrapped in list() to materialize.

filter() — keep matching elements
amounts = [1500.00, 250.75, 4500.00, 89.99]

large_txns = list(filter(lambda a: a > 1000, amounts))

print(large_txns)   # Output: [1500.0, 4500.0]

sorted(iterable, key=, reverse=) returns a new sorted list from any iterable, leaving the source untouched. The key parameter sorts by a computed value.

sorted() — new sorted list, with key=
products = [("Mouse", 29.99), ("Monitor", 199.00), ("Keyboard", 49.99)]

by_price = sorted(products, key=lambda p: p[1], reverse=True)

print(by_price)
# Output: [('Monitor', 199.0), ('Keyboard', 49.99), ('Mouse', 29.99)]

reversed(sequence) iterates a sequence backwards without copying or modifying it.

reversed() — iterate backwards
steps = ["extract", "transform", "load"]

for step in reversed(steps):
    print(step)

# Output:
# load
# transform
# extract

Boolean Aggregators

any(iterable) returns True if at least one element is truthy — perfect for "did anything fail?" checks.

any() — is at least one True?
failed_checks = [False, False, True, False]

if any(failed_checks):
    print("At least one check failed — send alert")

# Output: At least one check failed — send alert
print(any([False, False]))   # Output: False

all(iterable) returns True only when every element is truthy — the natural quality gate. Note: all([]) is True (an empty checklist "passes").

all() — is every element True?
qa_checks = [True, True, True, False]

if all(qa_checks):
    print("Pipeline approved for production")
else:
    print("Blocked — review failures")

# Output: Blocked — review failures
print(all([]))   # Output: True — empty iterable passes!

Input & Output

input(prompt) pauses the program, shows a prompt, and returns whatever the user types — always as a string, so cast before doing math.

input() — read from the user (always str)
raw = input("Enter withdrawal amount: ")   # user types: 500

amount = float(raw)          # cast before arithmetic
print(amount * 2)            # Output: 1000.0
print(raw * 2)               # Output: 500500 — string repetition without the cast!

print() accepts multiple values plus two formatting parameters: sep (between values) and end (after the last one).

print() — sep and end options
print("ACC-8842", "DEBIT", 1500.0, sep=" | ")
# Output: ACC-8842 | DEBIT | 1500.0

print("Processing", end=" ")
print("complete.")
# Output: Processing complete.

Edge Cases & Gotchas

map() and filter() return one-shot iterators: once consumed, they are empty. Convert to a list if you need the results more than once.

Gotcha — map/filter iterators are single-use
amounts = map(float, ["10", "20"])

print(sum(amounts))     # Output: 30.0 — iterator consumed here
print(sum(amounts))     # Output: 0 — nothing left!

amounts = list(map(float, ["10", "20"]))   # materialize once
print(sum(amounts), sum(amounts))            # Output: 30.0 30.0

Real-World Application

  • ETL — map(float, ...) casts CSV columns; zip() reunites header and row lists into dicts.
  • Quality gates — all(checks) blocks deployment; any(failures) triggers alerts.
  • Reporting — sorted(..., key=..., reverse=True) plus slicing produces every top-N ranking.

Notebook & Practice

  • Combine parallel ID and revenue lists into a dict with zip().
  • Print a numbered menu with enumerate(..., start=1).
  • Cast a list of amount strings with map(float, ...), then filter those above 1000.
  • Sort a list of (name, price) tuples by price with a key function.
  • Verify all() and any() on an empty list and explain the results.

Key Takeaways

  • len, type, isinstance inspect data; prefer isinstance for type checks.
  • range, enumerate, zip power clean loops without manual counters.
  • map/filter transform lazily and are single-use; sorted/reversed never touch the original.
  • any/all compress validation loops into one readable call — and all([]) is True.
  • input() always returns a string; print() takes sep and end.
