Loop Control

Subtopics: break statement, continue statement, pass statement, else clause in loops

Steering a Running Loop

Scanning 50,000 records does not mean touching all 50,000 when the match appears at row 12. Loop-control statements change a loop's flow from inside: break exits immediately, continue skips to the next iteration, pass does nothing (a placeholder), and the rarely-taught else clause runs only when the loop finished without breaking.

Theory & Concepts

StatementEffectTypical Use
breakExit the loop nowStop searching after the first match
continueSkip the rest of this iterationFilter out bad rows without nesting
passDo nothingStub out a block you will write later
loop elseRun only if no break occurred"Searched everything, found nothing"

In nested loops, break and continue affect only the innermost loop containing them.

Syntax Reference

break abandons the loop the moment its job is done — the remaining elements are never visited.

break — stop at the first match
txn_ids = ["TXN-001", "TXN-002", "TXN-FRAUD", "TXN-004"]

for txn_id in txn_ids:
    if txn_id == "TXN-FRAUD":
        print(f"Fraud found: {txn_id}")
        break                          # TXN-004 is never checked
    print(f"Checked {txn_id}")

# Output:
# Checked TXN-001
# Checked TXN-002
# Fraud found: TXN-FRAUD

continue skips only the current element — the loop itself keeps going. It filters bad data without wrapping the whole body in an if.

continue — skip invalid rows
amounts = [150.0, -5.0, 200.0, 0.0, 89.0]
total = 0

for amt in amounts:
    if amt <= 0:
        continue              # bad data — jump to the next amount
    total += amt              # only valid amounts reach this line

print(f"Valid sales total: ${total}")
# Output: Valid sales total: $439.0

pass is a syntactic placeholder: Python requires something in every block, and pass satisfies that while doing nothing — useful while sketching a pipeline.

pass — an intentional no-op
order_statuses = ["shipped", "pending", "returned"]

for status in order_statuses:
    if status == "returned":
        pass                  # TODO: refund logic goes here later
    else:
        print(f"Processing {status} order")

# Output:
# Processing shipped order
# Processing pending order

The loop else runs exactly when the loop ran to completion — that is, no break fired. It expresses "searched everything and did not find it" without a flag variable.

Loop else — search completed, no match
accounts = ["ACC-101", "ACC-102", "ACC-103"]
search = "ACC-999"

for acc in accounts:
    if acc == search:
        print("Found")
        break
else:                             # belongs to the FOR, not an if
    print("Account not found")

# Output: Account not found

Edge Cases & Gotchas

In nested loops, break only exits the innermost loop — the outer one continues. Escaping both usually means restructuring into a function with return.

Gotcha — break only exits one level
stores = ["S1", "S2"]
skus = ["OK-1", "BAD", "OK-2"]

for store in stores:
    for sku in skus:
        if sku == "BAD":
            break                 # exits the sku loop only
        print(f"{store}: {sku}")

# Output:
# S1: OK-1
# S2: OK-1
# The outer loop still ran for S2 — break did not stop it.

Real-World Application

  • Search — break ends a scan the moment the target record is found, saving compute on large batches.
  • Data quality — continue drops cancelled or malformed rows while the valid ones aggregate.
  • Verification — loop else reports "ID not in roster" cleanly after an exhaustive search.

Notebook & Practice

  • Scan a list for a target value and break on the first hit; count how many checks were saved.
  • Sum only positive amounts using continue.
  • Stub a branch with pass and a TODO comment.
  • Search for a missing ID and report it via the loop's else clause.
  • Show that break in a nested loop leaves the outer loop running.

Key Takeaways

  • break exits the loop; continue skips to the next iteration; both affect only the innermost loop.
  • pass is a do-nothing placeholder that keeps stub blocks syntactically valid.
  • The loop else runs only when no break fired — ideal for not-found reporting.
  • Early exits and skips make scans faster and filtering code flatter.
