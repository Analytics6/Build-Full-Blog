String Functions

Clean, format, and search text with CONCAT, SUBSTR, UPPER, LOWER, and TRIM — essential for matching normalizing job titles and employee name fields.

Text Cleaning in Pipelines

Source systems export text fields with inconsistent casing; rows may store values with trailing spaces. Before JOINs and GROUP BYs work reliably, analysts normalize strings in SQL. String functions let you transform text inside the database without exporting to Python first.

Theory & Concepts

Syntax varies slightly by database: PostgreSQL uses SUBSTRING or SUBSTR; SQL Server uses SUBSTRING; SQLite supports all common forms. examples use portable syntax where payrollsible.

Example: Normalize Product Data

SQL — Normalize text columns
SELECT
    job_id,
    UPPER(TRIM(job_id)) AS sku_normalized,
    INITCAP(LOWER(TRIM(job_title))) AS product_name_clean,
    SUBSTR(job_id, 1, 3) AS category_code
FROM hr.jobs
WHERE LENGTH(TRIM(job_id)) >= 5;
SQL — Build a searchable label
SELECT
    job_id,
    CONCAT(job_id, ' — ', job_title, ' (', category, ')') AS shelf_label
FROM hr.jobs
ORDER BY category, job_title;

Example: Parse and Match Account Data

SQL — Case-insensitive lookup
SELECT
    employee_id,
    account_number,
    last_name
FROM hr.employees
WHERE LOWER(TRIM(last_name)) = LOWER('  Priya Sharma  ');
SQL — Extract branch code from transaction reference
-- Reference format: TXN-BR042-20260716-001
SELECT
    employee_id,
    email,
    SUBSTR(email, 5, 5) AS branch_code,
    REPLACE(email, '-', '') AS reference_compact
FROM hr.job_history
WHERE email LIKE 'TXN-%';

Real-World Notes

  • Fuzzy matching — LOWER(TRIM(col)) on both sides of a JOIN handles casing and whitespace mismatches between the employees table and the jobs table.
  • Performance — Applying functions on indexed columns prevents index use. Clean data at ingest time and store a normalized column.
  • LIKE patterns — Combine UPPER(name) LIKE '%ELECTRON%' for flexible job title searches; prefer full-text indexes at scale.
  • PII handling — Mask account numbers in reports: SUBSTR(employee_id, 1, 4) || '****' || SUBSTR(employee_id, -4).

Key Takeaways

  • String functions clean and reshape text without leaving SQL.
  • Always TRIM imported names and codes before comparing or joining.
  • UPPER/LOWER enable case-insensitive matching.
  • SUBSTR and REPLACE parse fixed-format HR identifiers.
