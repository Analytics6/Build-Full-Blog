Conditional Statements

Subtopics: if statement, else statement, elif chain, conditional logic

Go to Assignment →

Programs That Decide

Everything so far ran top to bottom, every line, every time. Conditional statements change that: if, elif, and else let a program choose between paths based on boolean conditions. Is the amount above the threshold? Take the fraud path. Is stock below the reorder point? Take the reorder path. Branching is where code starts encoding real business rules.

Theory & Concepts

How a Chain Evaluates

  • Conditions are checked top to bottom; the first true branch runs and all the rest are skipped — even if later conditions would also be true.
  • elif (else-if) adds more mutually exclusive branches; else catches everything that matched nothing.
  • elif and else are optional; a bare if is fine.
  • Each block after a colon is indented 4 spaces — the indentation is the block.

Best practice: order conditions from most specific to most general. A chain that checks > 10000 before > 50000 will never reach the second branch — the first one swallows everything.

Syntax Reference

The simplest form: a single if with an optional else covering the two-way decision.

if / else — a two-way decision
stock_qty = 8
reorder_point = 15

if stock_qty <= reorder_point:
    print("Reorder SKU from supplier")
else:
    print("Stock sufficient")

# Output: Reorder SKU from supplier

An elif chain handles three or more mutually exclusive outcomes. Exactly one branch runs — the first whose condition is true.

if / elif / else — risk tiers
transaction_amount = 95000

if transaction_amount > 50000:
    risk_level = "HIGH"          # matched first — chain stops here
elif transaction_amount > 10000:
    risk_level = "MEDIUM"        # would match too, but is never reached
else:
    risk_level = "LOW"

print(f"Risk: {risk_level}")     # Output: Risk: HIGH

Longer chains grade a value into buckets. Because evaluation stops at the first match, thresholds must be ordered from highest to lowest.

Grading buckets — ordered thresholds
orders_completed = 15
orders_total = 20
completion = orders_completed / orders_total     # 0.75

if completion >= 1.0:
    status = "Complete"
elif completion >= 0.75:
    status = "On Track"
elif completion >= 0.50:
    status = "Behind"
else:
    status = "At Risk"

print(f"Status: {status}")   # Output: Status: On Track

The one-line conditional expression (ternary) fits simple either/or assignments without a full block.

Conditional expression — one-line if/else
account_balance = 15420.50

tier = "premium" if account_balance >= 10000 else "standard"

print(tier)   # Output: premium

Edge Cases & Gotchas

Wrongly ordered thresholds are the classic conditional bug: the general case swallows the specific one, and no error is raised.

Gotcha — condition order matters
amount = 95000

# WRONG order: the broad condition catches everything first
if amount > 10000:
    label = "MEDIUM"       # 95000 > 10000, so this wins...
elif amount > 50000:
    label = "HIGH"         # ...and this is unreachable!

print(label)   # Output: MEDIUM — silently wrong

Real-World Application

  • ETL routing — records branch to different transform functions based on a source-system field.
  • Segmentation — customers fall into spend buckets for targeted offers via ordered elif chains.
  • Approvals — loan applications pass or fail on credit-score thresholds encoded exactly like the grading example.

Notebook & Practice

  • Write an if/else that prints whether a stock quantity needs reordering.
  • Build a four-branch elif chain grading a completion percentage.
  • Rewrite a simple two-way decision as a one-line conditional expression.
  • Deliberately reverse two thresholds and observe the silent wrong answer.

Key Takeaways

  • if / elif / else — exactly one branch executes; evaluation stops at the first true condition.
  • Order conditions from most specific (highest threshold) to most general.
  • Indentation defines each block; every branch body needs 4 spaces.
  • Use the ternary form a if cond else b for simple assignments only.
  • Misordered chains fail silently — test each branch with a sample value.
