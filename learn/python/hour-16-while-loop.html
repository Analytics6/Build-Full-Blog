While Loop

Subtopics: while loop syntax, infinite loops, loop conditions, updating state variables

Go to Assignment →

Repeat Until Something Changes

Not every repetition has a known count. A retry loop runs until the connection succeeds; an input loop runs until the user types "exit"; a balance drains until it hits zero. The while loop repeats its body as long as a condition stays True — indefinite iteration, where your code decides at runtime when to stop. The price of that flexibility: you are responsible for making the condition eventually turn false.

Theory & Concepts

Evaluation Order

while condition: checks the condition before each pass. If it is false at the very start, the body never runs at all. After each pass, the condition is checked again.

The Three-Part Recipe

  • Initialize a state variable before the loop.
  • Test that variable in the condition.
  • Update it inside the body — forget this and the loop runs forever.

for vs while

Iterating over a known collection or count → for. Repeating until an unpredictable event (user input, API success, threshold reached) → while. If you can write it as a for, do — it is harder to get wrong.

Syntax Reference

The canonical counter loop: state initialized, tested, and updated.

while — batch progress counter
processed = 0                        # 1. initialize
batch_size = 1000
total_txns = 4500

while processed < total_txns:        # 2. test
    processed += batch_size          # 3. update
    print(f"Processed {min(processed, total_txns)} of {total_txns}")

# Output:
# Processed 1000 of 4500
# Processed 2000 of 4500
# Processed 3000 of 4500
# Processed 4000 of 4500
# Processed 4500 of 4500

A countdown makes the update direction explicit — subtract until the guard fails.

while — retry countdown
retries = 3

while retries > 0:
    print(f"Attempting warehouse connection... ({retries} left)")
    retries -= 1                     # MUST decrease, or the loop never ends

print("Out of retries")

# Output:
# Attempting warehouse connection... (3 left)
# Attempting warehouse connection... (2 left)
# Attempting warehouse connection... (1 left)
# Out of retries

Interactive loops run until a sentinel value arrives — the classic point-of-sale pattern.

while — input until a sentinel
command = ""

while command != "exit":
    command = input("Enter SKU or 'exit': ").strip()
    if command != "exit":
        print(f"Looking up {command} in product_catalog...")

print("Session closed")

# Example session:
# Enter SKU or 'exit': SKU-42
# Looking up SKU-42 in product_catalog...
# Enter SKU or 'exit': exit
# Session closed

A condition that is false from the start means zero iterations — sometimes exactly what you want, sometimes a sign of an inverted comparison.

while — a false condition never runs
overdue_amount = 0

while overdue_amount > 0:
    print("Sending payment reminder")      # never executes
    overdue_amount -= 100

print("No reminders needed")
# Output: No reminders needed

Edge Cases & Gotchas

The infinite loop: forget the update (or update in the wrong direction) and the program hangs, consuming CPU until killed. If a running cell never finishes, this is the first thing to check.

Gotcha — the infinite loop
balance = 500

# WRONG — balance never changes, condition stays True forever:
# while balance > 0:
#     print("Charging fee")          <-- runs forever; Ctrl+C / interrupt kernel

# CORRECT — the state moves toward the exit:
while balance > 0:
    balance -= 200                    # fee reduces the balance
    print(f"Fee charged. Remaining: {balance}")

# Output:
# Fee charged. Remaining: 300
# Fee charged. Remaining: 100
# Fee charged. Remaining: -100

Real-World Application

  • Resilience — retry loops around flaky network calls, capped by a max-attempts counter.
  • Polling — monitors re-check a queue or table until a threshold is met.
  • Interactive tools — CLIs accept commands until the user types a sentinel like "exit".

Key Takeaways

  • while re-checks its condition before every pass; a false start means zero iterations.
  • Initialize, test, update — miss the update and the loop is infinite.
  • Prefer for for known sequences; reserve while for open-ended repetition.
  • Sentinel-driven input loops and capped retry loops are the two everyday patterns.
