Working with Dicts Safely

Bracket access crashes on missing keys, and plain loops only see keys. Dictionary methods fill the gaps: safe reads with defaults, clean iteration over pairs, merging, and controlled removal. In production pipelines these methods are the difference between a script that survives messy input and one that dies on the first incomplete record.

Viewing Contents

keys() returns a live view of all keys — loop over it, or convert to a list.

dict.keys() — all keys
txn = {"transaction_id": "TXN-8842", "amount": 1500.00, "txn_type": "DEBIT"}

print(list(txn.keys()))
# Output: ['transaction_id', 'amount', 'txn_type']

values() returns the matching view of all values — useful for totals and quick scans.

dict.values() — all values
daily_totals = {"Mon": 1250, "Tue": 980, "Wed": 1450}

print(list(daily_totals.values()))   # Output: [1250, 980, 1450]
print(sum(daily_totals.values()))    # Output: 3680 — weekly total so far

items() yields (key, value) pairs — the idiomatic way to loop over a dict when you need both parts.

dict.items() — key-value pairs
prices = {"Mouse": 29.99, "Keyboard": 49.99}

for name, price in prices.items():
    print(f"{name}: ${price}")

# Output:
# Mouse: $29.99
# Keyboard: $49.99

Safe Access

get(key, default) reads a key without risking KeyError — it returns the default (or None) when the key is absent. The workhorse of defensive data code.

dict.get() — read with a fallback
etl_config = {"source": "transactions.csv", "batch_size": 1000}

print(etl_config.get("batch_size", 500))    # Output: 1000 — key exists
print(etl_config.get("timeout_sec", 30))    # Output: 30 — default used
print(etl_config.get("timeout_sec"))        # Output: None — default default

setdefault(key, default) reads like get(), but it also inserts the default into the dict when the key is missing. Existing keys are never overwritten.

dict.setdefault() — insert only if missing
account = {"account_id": "ACC-1001"}

account.setdefault("status", "ACTIVE")       # missing — inserted
account.setdefault("account_id", "NEW")      # exists — untouched

print(account)
# Output: {'account_id': 'ACC-1001', 'status': 'ACTIVE'}

Merging & Removing

update(other) merges another dict in place: new keys are added, existing keys are overwritten by the incoming values.

dict.update() — merge dicts
profile = {"name": "Priya", "email": "priya@bank.com"}
changes = {"phone": "9876543210", "email": "priya.new@bank.com"}

profile.update(changes)

print(profile)
# Output: {'name': 'Priya', 'email': 'priya.new@bank.com', 'phone': '9876543210'}
# The email was overwritten; the phone was added.

pop(key, default) removes a key and returns its value. Pass a default to avoid KeyError on missing keys.

dict.pop() — remove and return
txn = {"transaction_id": "TXN-8842", "temp_score": 0.87}

score = txn.pop("temp_score")          # removed AND returned
limit = txn.pop("credit_limit", 0)     # missing — default returned

print(score)   # Output: 0.87
print(limit)   # Output: 0
print(txn)     # Output: {'transaction_id': 'TXN-8842'}

popitem() removes and returns the last inserted pair as a tuple — occasionally useful for draining a dict.

dict.popitem() — remove the last pair
queue = {"job_1": "done", "job_2": "pending"}

last_pair = queue.popitem()

print(last_pair)   # Output: ('job_2', 'pending')
print(queue)       # Output: {'job_1': 'done'}

clear() removes every pair, leaving an empty dict.

dict.clear() — empty the dict
session_cache = {"user": "priya", "expires": "17:00"}

session_cache.clear()

print(session_cache)   # Output: {}

Copying & Building

copy() makes a shallow copy so you can experiment without touching the original — same caveat as lists: nested objects are shared.

dict.copy() — independent shallow copy
base_config = {"batch_size": 1000, "retries": 3}

test_config = base_config.copy()
test_config["batch_size"] = 50        # tweak the copy only

print(base_config)   # Output: {'batch_size': 1000, 'retries': 3}
print(test_config)   # Output: {'batch_size': 50, 'retries': 3}

dict.fromkeys(keys, value) builds a new dict where every key starts with the same value — handy for initializing counters.

dict.fromkeys() — initialize many keys
stores = ["S1", "S2", "S3"]

sales_counters = dict.fromkeys(stores, 0)

print(sales_counters)
# Output: {'S1': 0, 'S2': 0, 'S3': 0}

A dict comprehension builds a dict from any iterable in one line — the dictionary sibling of the list comprehension.

Dict comprehension — build in one line
prices = {"Mouse": 29.99, "Keyboard": 49.99, "Monitor": 199.00}

# Apply a 10% discount to every product
discounted = {name: round(p * 0.9, 2) for name, p in prices.items()}
print(discounted)
# Output: {'Mouse': 26.99, 'Keyboard': 44.99, 'Monitor': 179.1}

# Filter: keep only items under $50
budget = {name: p for name, p in prices.items() if p < 50}
print(budget)
# Output: {'Mouse': 29.99, 'Keyboard': 49.99}

Edge Cases & Gotchas

Choose access styles deliberately: brackets crash loudly (good for required fields), get() degrades quietly (good for optional fields). Using get() everywhere can hide real data problems.

Gotcha — [ ] vs get(): loud vs quiet failures
row = {"transaction_id": "TXN-8842"}    # amount is missing!

amount = row.get("amount", 0)            # quiet — a fake $0 enters the report
print(amount)                             # Output: 0

# amount = row["amount"]                 <-- KeyError — loud, but the bug is visible
# Rule of thumb: required field -> brackets; optional field -> get()

Real-World Application

  • Configs — config.get("DB_PORT", 5432) supplies safe defaults; update() layers environment overrides on top of base settings.
  • Aggregation — fromkeys() or setdefault() initialize per-store counters before a tally loop.
  • Transformation — dict comprehensions reshape API payloads (rename keys, convert units) in one pass.

  • Merge a changes dict into a profile with update() and note which values were overwritten.
  • Initialize counters for three stores with fromkeys(), then increment one.
  • Write a dict comprehension applying a discount, and another that filters by price.

Key Takeaways

  • keys(), values(), items() — views for iteration; items() unpacks into for k, v in ....
  • get() reads safely with a default; setdefault() additionally inserts the default.
  • update() merges (incoming wins); pop()/popitem()/clear() remove with increasing scope.
  • copy() is shallow; fromkeys() initializes many keys at once.
  • Dict comprehensions transform and filter dicts in a single readable line.
