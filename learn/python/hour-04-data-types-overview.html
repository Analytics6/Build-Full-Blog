Data Types Overview

Subtopics: int, float, str, bool, type() function

Go to Assignment →

Why Types Matter

Every value in Python has a type, and the type determines what you can do with it. You can multiply two numbers but not two names; you can uppercase a string but not an integer. When data arrives from the outside world — a CSV file, an API response, user input — it often arrives as text, and converting it to the right type is one of the most common (and most commonly forgotten) steps in data work.

Theory & Concepts

The Four Primitive Types

TypeExampleTypical Use
int42, -7, 0Counts, IDs, quantities
float3.14, -0.5Prices, amounts, rates, percentages
str"Wireless Mouse", 'ACC-001'Names, codes, any text
boolTrue, FalseFlags, yes/no states, conditions

Note that bool is actually a subtype of int — True behaves as 1 and False as 0. That is why True + True equals 2, and why sum() over a list of booleans counts the True values — a genuinely useful trick in data analysis.

Checking Types with type()

type(value) returns the class of any object. Reach for it whenever a calculation misbehaves — the most frequent cause is a number that is secretly a string.

Type Casting

Convert between types explicitly with the constructor functions int(), float(), str(), and bool(). Two rules to remember:

  • int(49.99) truncates toward zero — it gives 49, it does not round. Use round() if you want rounding.
  • int("49.99") raises ValueError — a string must look like a plain integer to cast directly. Go through float() first.

Syntax Reference

Example — Inspecting types of transaction fields
# Fields from one banking transaction record
customer_id = 8842              # int — a whole-number identifier
loan_amount = 250000.00         # float — money needs decimals
account_type = "SAVINGS"        # str — a text code
is_active = True                # bool — a yes/no flag

print(type(customer_id))    # Output: <class 'int'>
print(type(loan_amount))    # Output: <class 'float'>
print(type(account_type))   # Output: <class 'str'>
print(type(is_active))      # Output: <class 'bool'>
Example — Casting CSV strings before math
# CSV columns arrive as text — cast them before any arithmetic
raw_amount = "1549.99"            # str straight from a store_sales CSV
raw_qty = "12"

sale_amount = float(raw_amount)    # now usable in calculations
quantity = int(raw_qty)

line_total = sale_amount * quantity
print(f"Line total: {line_total:.2f}")   # Output: Line total: 18599.88

# Forgetting the cast produces silent nonsense, not an error:
print(raw_qty * 3)   # Output: 121212 — string repetition, not arithmetic!

int() applied to a float truncates — it simply drops the decimal part. Use round() when you want normal rounding.

Example — int() truncates, round() rounds
print(int(49.99))    # Output: 49 — decimals dropped
print(round(49.99))  # Output: 50 — properly rounded
print(int(-49.99))   # Output: -49 — truncates toward zero

str() converts numbers to text — required before concatenating them with other strings.

Example — str() for building labels
customer_num = 20481

customer_label = "CUS-" + str(customer_num)   # cast before concatenation
print(customer_label)   # Output: CUS-20481

# "CUS-" + customer_num  <-- TypeError: can only concatenate str to str

bool() follows the truthiness rules: zero and empty values convert to False, everything else to True — including the text "False".

Example — bool() and truthiness
print(bool(0))          # Output: False — zero
print(bool(""))         # Output: False — empty string
print(bool("fraud"))    # Output: True — any non-empty string
print(bool("False"))    # Output: True — beware! non-empty text is truthy
Example — Counting True flags with sum()
# Because bool is a subtype of int, True counts as 1
fraud_flags = [True, False, True, True, False]

flagged_count = sum(fraud_flags)   # adds 1 for every True
print(f"Flagged transactions: {flagged_count}")
# Output: Flagged transactions: 3

Real-World Application

  • ETL — explicit casting prevents an amount column of strings from breaking SUM aggregations after a CSV load.
  • Data quality checks — type() assertions catch schema drift when an upstream feed changes a column from int to string.
  • Analytics — boolean columns like is_fraud can be summed directly to count positives, thanks to bool being an int subtype.

Key Takeaways

  • The four primitives are int, float, str, and bool; every value has exactly one type.
  • type() is your first debugging tool when arithmetic misbehaves.
  • External data (CSVs, input, APIs) arrives as strings — cast explicitly before doing math.
  • int() truncates; round() rounds; int("49.99") raises an error.
  • Non-empty strings are always truthy — even "False".
