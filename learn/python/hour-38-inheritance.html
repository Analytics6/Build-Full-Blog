Inheritance

Subtopics: Parent vs child class, super() function, method overriding, polymorphism basics

Go to Assignment →

Why Inheritance Matters in Data Analytics

Organizations model many entity types that share common traits: every customer has a name and ID; savings and checking accounts both hold balances but behave differently. Inheritance lets a child class reuse a parent class's attributes and methods, then customize or extend behavior — reducing duplication across industries domain models.

Theory & Concepts

Parent and Child Classes

  • Parent (superclass) — general blueprint: class Person
  • Child (subclass) — specialized version: class SavingsAccount(BankAccount)

super()

Calls the parent class constructor or methods: super().__init__(name) ensures parent attributes are initialized before adding child-specific ones.

Method Overriding

Define a method in the child with the same name as the parent to replace behavior. The child version runs when called on child instances.

Syntax Reference

The basic inheritance syntax: class Child(Parent). The child automatically gains every attribute and method the parent defines.

Inheritance — child reuses parent behavior
class BankAccount:
    def __init__(self, account_id, balance):
        self.account_id = account_id
        self.balance = balance

    def describe(self):
        return f"Account {self.account_id}: ${self.balance}"

class SavingsAccount(BankAccount):    # inherits everything
    pass

acct = SavingsAccount("ACC-8842", 1500.00)
print(acct.describe())    # Output: Account ACC-8842: $1500.0 — parent method works

super().__init__() runs the parent constructor first, so the child can add its own attributes on top of a fully initialized base.

super() — extend the parent constructor
class BankAccount:
    def __init__(self, account_id, balance):
        self.account_id = account_id
        self.balance = balance

class SavingsAccount(BankAccount):
    def __init__(self, account_id, balance, interest_rate):
        super().__init__(account_id, balance)    # parent sets id + balance
        self.interest_rate = interest_rate       # child adds its own field

acct = SavingsAccount("ACC-8842", 1500.00, 0.04)

print(acct.balance)          # Output: 1500.0 — from the parent
print(acct.interest_rate)    # Output: 0.04 — from the child

Method overriding: define a method with the parent's name in the child, and the child's version wins for child instances.

Overriding — replace parent behavior
class BankAccount:
    def monthly_fee(self):
        return 10.00

class SavingsAccount(BankAccount):
    def monthly_fee(self):          # override — savings accounts are free
        return 0.00

regular = BankAccount()
savings = SavingsAccount()

print(regular.monthly_fee())    # Output: 10.0
print(savings.monthly_fee())    # Output: 0.0 — child version wins

An override can still reuse the parent's logic by calling super().method() — extend rather than replace.

super() in overrides — extend, not replace
class Vehicle:
    def __init__(self, brand):
        self.brand = brand

    def start_engine(self):
        return f"{self.brand} engine started"

class DeliveryVan(Vehicle):
    def start_engine(self):
        parent_msg = super().start_engine()      # run the parent version first
        return f"{parent_msg} — ready for delivery route"

van = DeliveryVan("Toyota")
print(van.start_engine())
# Output: Toyota engine started — ready for delivery route

Polymorphism: code that calls a shared method works with any subclass — each object responds in its own way.

Polymorphism — one call, many behaviors
accounts = [BankAccount(), SavingsAccount()]

for acct in accounts:
    print(acct.monthly_fee())    # same call, class-specific answer

# Output:
# 10.0
# 0.0

Real-World Application

  • Payment types — Base Payment class; CreditCardPayment and BankTransfer override process().
  • ETL validators — Base RecordValidator; child classes add domain-specific validation rules.
  • Framework patterns — Django model inheritance for shared timestamp fields on all tables.

Key Takeaways

  • Inheritance reuses parent attributes and methods in child classes.
  • super() initializes or calls parent behavior from the child.
  • Method overriding customizes behavior while keeping a shared interface.
  • Hierarchies model "is-a" relationships: a SavingsAccount is a BankAccount.
