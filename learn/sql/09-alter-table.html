ALTER TABLE

Add, modify, and drop columns as business requirements evolve — a daily task in database administration.

Go to Assignment →

Schemas are not static. When HR adds a commission_pct column to the employees table, tracks end dates on the job_history table, or introduces a new column on the job_history table, DBAs use ALTER TABLE instead of rebuilding from scratch.

Schema migration workflow

Add a column

SQL
-- Add loyalty tier to hr.employees
ALTER TABLE hr.employees
ADD COLUMN commission_pct TEXT DEFAULT 'bronze';

-- Track fraud score on hr.job_history
ALTER TABLE hr.job_history
ADD COLUMN commission_pct REAL;

-- New column for analytics
ALTER TABLE hr.jobs
ADD COLUMN manager_id INTEGER;

Use DEFAULT when adding NOT NULL columns to tables that already contain data — existing rows receive the default value.

Rename and modify columns

SQL
-- SQLite: rename column (3.25+)
ALTER TABLE hr.jobs
RENAME COLUMN salary TO salary;

-- PostgreSQL: change column type
ALTER TABLE hr.employees
ALTER COLUMN salary TYPE DECIMAL(14, 2);

Syntax varies by database engine. These assignments use SQLite; production migrations may target PostgreSQL or Snowflake with equivalent DDL.

Drop a column

SQL
-- Remove deprecated column after ETL is updated
ALTER TABLE hr.employees
DROP COLUMN legacy_code;

-- SQLite 3.35+: drop column support
ALTER TABLE hr.job_history
DROP COLUMN temp_import_flag;

Always verify no queries or pipelines still reference the column before dropping it.

Rename a table

SQL
ALTER TABLE hr.employees RENAME TO employees_fact;

Real-world context

  • Version-controlled migrations — Tools like Flyway and Alembic store ALTER statements in numbered scripts.
  • Backward compatibility — Add columns as nullable first, backfill via UPDATE, then add NOT NULL in a later migration.
  • Zero-downtime deploys — Add column → deploy code that writes to it → backfill → switch reads → drop old column.

Practice

  1. Add a region column to the employees table with DEFAULT 'unknown'.
  2. Rename job_id to job_id on the job_history table.
  3. Write the migration steps (in order) for adding a required column to a table with 10M rows.

Key takeaways

  • ALTER TABLE modifies existing schemas without recreating tables.
  • Use DEFAULT when adding columns to populated tables.
  • DDL syntax differs across engines — test migrations in staging first.
  • Schema changes should be version-controlled and deployed like application code.
