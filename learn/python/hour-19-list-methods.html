List Methods

Subtopics: append(), insert(), extend(), remove(), pop(), clear(), index(), count(), sort(), sorted(), reverse(), reversed(), copy()

Go to Assignment →

Growing, Shrinking, Reordering

Real datasets change constantly: new records arrive, bad rows get dropped, results get sorted for reports. List methods handle all of it. One crucial pattern to internalize: most list methods work in place and return None — writing data = data.sort() silently destroys your list. This hour covers each method individually, including the in-place vs copy distinction.

Adding Elements

append(item) adds a single item to the end of the list — the most used list method in Python, and the standard way to build a list inside a loop.

list.append() — add one item to the end
cart = ["SKU-0042"]

cart.append("SKU-0018")
cart.append("SKU-0077")

print(cart)   # Output: ['SKU-0042', 'SKU-0018', 'SKU-0077']

insert(index, item) places an item at a specific position, shifting everything after it to the right.

list.insert() — add at a position
cart = ["SKU-0042", "SKU-0018"]

cart.insert(0, "SKU-0099")     # priority item goes first

print(cart)   # Output: ['SKU-0099', 'SKU-0042', 'SKU-0018']

extend(iterable) appends every element of another iterable. Compare with append(), which would add the whole list as a single nested element — a classic mix-up.

list.extend() — merge another list
morning_txns = ["TXN-001", "TXN-002"]
evening_txns = ["TXN-003", "TXN-004"]

morning_txns.extend(evening_txns)
print(morning_txns)
# Output: ['TXN-001', 'TXN-002', 'TXN-003', 'TXN-004']

# append() would nest instead:
batch = ["TXN-001"]
batch.append(["TXN-002", "TXN-003"])
print(batch)   # Output: ['TXN-001', ['TXN-002', 'TXN-003']] — nested!

Removing Elements

remove(value) deletes the first occurrence of a value. If the value is absent it raises ValueError, so check membership first when unsure.

list.remove() — delete by value
customer_ids = ["C-101", "C-102", "C-101", "C-103"]

customer_ids.remove("C-101")    # only the FIRST C-101 goes

print(customer_ids)   # Output: ['C-102', 'C-101', 'C-103']
# customer_ids.remove("C-999")  <-- ValueError: not in list

pop(index) removes an item by position and returns it — the only removal method that gives the value back. With no argument it pops the last item.

list.pop() — remove by index and return it
queue = ["TXN-001", "TXN-002", "TXN-003"]

last = queue.pop()        # no index = last item
first = queue.pop(0)      # index 0 = first item

print(last)     # Output: TXN-003
print(first)    # Output: TXN-001
print(queue)    # Output: ['TXN-002']

clear() empties the list in place, leaving an empty list rather than deleting the variable.

list.clear() — empty the list
batch = ["TXN-001", "TXN-002", "TXN-003"]

batch.clear()             # after a successful bulk insert

print(batch)       # Output: []
print(len(batch))  # Output: 0

Querying

index(value) returns the position of the first match — or raises ValueError if the value is missing (like str.index()).

list.index() — find a position
skus = ["SKU-001", "SKU-002", "SKU-003"]

print(skus.index("SKU-002"))    # Output: 1
# skus.index("SKU-999")         <-- ValueError: not in list

count(value) returns how many times a value appears — a one-line duplicate detector.

list.count() — count occurrences
customer_ids = ["C-101", "C-102", "C-101", "C-103"]

print(customer_ids.count("C-101"))   # Output: 2 — duplicate found
print(customer_ids.count("C-999"))   # Output: 0 — no error for missing

Reordering

sort() orders the list in place (ascending by default, reverse=True for descending) and returns None. The original order is lost.

list.sort() — sort in place
amounts = [150, 320, 89, 450, 210]

amounts.sort()
print(amounts)               # Output: [89, 150, 210, 320, 450]

amounts.sort(reverse=True)
print(amounts)               # Output: [450, 320, 210, 150, 89]

The built-in sorted() function returns a new sorted list and leaves the original untouched — the safe choice when you still need the original order.

sorted() — sorted copy, original preserved
amounts = [150, 320, 89, 450, 210]

ranked = sorted(amounts, reverse=True)

print(ranked)     # Output: [450, 320, 210, 150, 89]
print(amounts)    # Output: [150, 320, 89, 450, 210] — unchanged

reverse() flips the current element order in place — it does not sort, it just mirrors the list.

list.reverse() — flip order in place
log_lines = ["09:00 open", "12:30 audit", "17:00 close"]

log_lines.reverse()          # newest first for display

print(log_lines)
# Output: ['17:00 close', '12:30 audit', '09:00 open']

The built-in reversed() returns an iterator over the list backwards without modifying it — wrap it in list() to materialize, or loop over it directly.

reversed() — reversed view, original preserved
log_lines = ["09:00 open", "12:30 audit", "17:00 close"]

for line in reversed(log_lines):
    print(line)

# Output:
# 17:00 close
# 12:30 audit
# 09:00 open

print(log_lines[0])   # Output: 09:00 open — original untouched

Copying

copy() creates a shallow copy — a new list with the same elements — so you can mutate the copy without touching the source. Equivalent to lst[:].

list.copy() — independent shallow copy
price_list = [29.99, 49.99, 15.50]

working = price_list.copy()
working[0] = 19.99             # discount applied to the copy only

print(price_list)   # Output: [29.99, 49.99, 15.5] — unchanged
print(working)      # Output: [19.99, 49.99, 15.5]

Edge Cases & Gotchas

The number-one list bug: in-place methods return None. Assigning their result throws your data away.

Gotcha — in-place methods return None
amounts = [150, 320, 89]

amounts = amounts.sort()     # WRONG — sort() returns None
print(amounts)               # Output: None — the list is gone!

# Correct: either mutate in place OR take sorted()'s return value
amounts = [150, 320, 89]
amounts.sort()               # option 1: in place, no assignment
ranked = sorted(amounts)     # option 2: new list from sorted()
print(amounts, ranked)       # Output: [89, 150, 320] [89, 150, 320]

Real-World Application

  • ETL — append() builds row buffers; extend() merges daily batches; clear() resets the buffer after each bulk insert.
  • Analytics — sorted(amounts, reverse=True)[:3] produces a top-3 report without disturbing the source data.
  • Data quality — count() flags duplicate customer IDs before they reach the database.

Key Takeaways

  • Add: append (one item), insert (at position), extend (merge iterable).
  • Remove: remove (by value, first match), pop (by index, returns the item), clear (empty all).
  • Query: index raises on missing; count returns 0 safely.
  • sort()/reverse() mutate in place and return None; sorted()/reversed() leave the original intact.
  • copy() (or [:]) makes an independent shallow copy for safe experimentation.
