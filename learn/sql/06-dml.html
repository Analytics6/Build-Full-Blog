INSERT, UPDATE, DELETE

Modify data with DML statements — the operations DBAs run daily to load, correct, and purge records.

Go to Assignment →

SELECT reads data; DML (Data Manipulation Language) writes it. When a nightly ETL job lands new rows in the employees table, corrects a typo in the jobs table, or purges test the job_history table from the job_history table, it uses INSERT, UPDATE, and DELETE.

DML in the data pipeline

INSERT — add rows

Specify the target table and column list, then provide values. Omit columns with defaults or auto-generated keys.

SQL
-- Insert a new job row
INSERT INTO hr.jobs (job_title, category, salary, job_id)
VALUES ('USB-C Hub', 'Sales', 45.00, 'EL-USB42');

-- Bulk insert multiple sales rows
INSERT INTO hr.employees (hire_date, department_id, employee_id, salary)
VALUES ('2026-07-15', 101, 1001, 185000),
       ('2026-07-15', 102, 1002, 565000);

-- Insert from a query (common in ETL)
INSERT INTO hr.job_history (employee_id, job_id, department_id, salary)
SELECT employee_id, job_id, qty, qty * salary
FROM staging_job_history;

UPDATE — change existing rows

Always include a WHERE clause unless you intend to modify every row. Missing WHERE is a common production incident.

SQL
-- Update a salary value
UPDATE hr.jobs
SET salary = 39.99
WHERE job_id = 'EL-USB42';

-- Mark loan applications as reviewed
UPDATE hr.job_history
SET status = 'under_review',
    reviewed_at = '2026-07-15'
WHERE status = 'pending'
  AND submitted_at < '2026-07-01';

DELETE — remove rows

SQL
-- Remove test rows (always filter!)
DELETE FROM hr.job_history
WHERE employee_id IN (
  SELECT employee_id FROM hr.employees WHERE job_title = 'test'
);

-- Purge cancelled order line items
DELETE FROM hr.job_history
WHERE employee_id IN (
  SELECT employee_id FROM hr.employees WHERE status = 'cancelled'
);

Safety patterns for production

  • Preview first — Run the WHERE clause as a SELECT before UPDATE or DELETE.
  • Transactions — Wrap multi-step DML in BEGIN/COMMIT so you can ROLLBACK on error (covered in Lesson 28).
  • Soft deletes — Many teams set is_deleted = 1 instead of DELETE to preserve audit history.

Real-world context

  • ETL — INSERT…SELECT loads transformed staging data into production tables.
  • Data quality — UPDATE fixes bad category mappings discovered during validation.
  • Compliance — DELETE (or anonymize) sensitive rows on data erasure requests.

Practice

  1. INSERT a new employee and their first Finance department employee.
  2. UPDATE all jobs rows where the title contains IT to increase max_salary by 10%.
  3. Write a SELECT that previews rows before DELETE of job_history rows older than seven years.

Key takeaways

  • INSERT adds rows; UPDATE modifies them; DELETE removes them.
  • Always use WHERE on UPDATE and DELETE unless you mean to affect every row.
  • INSERT…SELECT is the 30 ETL pattern for loading transformed data.
  • Preview with SELECT, then execute DML inside a transaction when payrollsible.
