Pandas Filtering

Subtopics: Selecting columns, selecting rows, conditional selection (Boolean indexing), loc (label-based) and iloc (integer position-based) indexers, combining multiple conditions (&, |)

Go to Assignment →

Filtering DataFrames Like SQL WHERE

Pandas boolean indexing mirrors SQL WHERE clauses. Filter store_sales for amounts above $1000, select customers with lifetime spend > $5,000, or isolate high-risk transactions — all with df[condition], .loc, and .iloc.

Theory & Concepts

Boolean Indexing

df[df["amount"] > 1000] keeps rows where the condition is True. Combine with &, |, ~ — wrap each condition in parentheses.

.loc vs .iloc

  • .loc — label-based selection by row/column names
  • .iloc — integer position-based selection (0-indexed)

Syntax Reference

Select one column with df["col"] (returns a Series) or several with a list of names (returns a DataFrame).

Selecting columns — one or several
amounts = df["amount"]                            # Series
pair = df[["transaction_id", "amount"]]          # DataFrame with 2 columns

print(type(amounts).__name__)    # Output: Series
print(type(pair).__name__)       # Output: DataFrame

Boolean indexing keeps only the rows where a condition is True — the pandas equivalent of SQL WHERE.

Boolean indexing — filter rows by condition
high_sales = df[df["amount"] > 1000]

print(len(df))            # Output (example): 14230 — all rows
print(len(high_sales))    # Output (example): 892 — rows above $1000

Combine conditions with & (and), | (or), and ~ (not). Each condition must be wrapped in parentheses — using Python's and/or here raises an error.

Combining conditions — & and |
electronics = df[(df["category"] == "Electronics") & (df["amount"] > 500)]
big_or_food = df[(df["amount"] > 5000) | (df["category"] == "Groceries")]

print(len(electronics))    # Output (example): 214
print(len(big_or_food))    # Output (example): 3105

.loc selects by labels: a row condition plus a list of column names in one step.

.loc — label-based selection
subset = df.loc[df["amount"] > 1000, ["transaction_id", "amount"]]

print(subset.head(2))
# Output (example):
#     transaction_id   amount
# 17          T-0018  1250.00
# 42          T-0043  3999.99

.iloc selects by integer position — row and column ranges follow the same exclusive-stop rule as list slicing.

.iloc — position-based selection
first_five = df.iloc[0:5, 0:3]    # rows 0-4, columns 0-2

print(first_five.shape)    # Output: (5, 3)

Real-World Application

  • Analytics dashboards — Filter products by price range and category for store managers.
  • Fraud detection — Isolate transactions above threshold in foreign countries.
  • reports — Extract customers with lifetime spend > $5,000 who shopped in Q4.

Notebook & Practice

  • Filter transactions above $1000 in the Electronics category.
  • Select rows 10—20 with Name and Price columns using .loc.
  • Grab the first 5 rows and 2 columns with .iloc.
  • Combine two conditions with & to filter by join year and balance.

Key Takeaways

  • df[df["col"] > val] — boolean indexing.
  • Use &, |, ~ (not and/or) for compound filters.
  • .loc = labels; .iloc = integer positions.
