Lists Basics

Subtopics: List creation, accessing elements (indexing), updating elements, list length, mutability

Go to Assignment →

Ordered, Changeable Collections

A list is Python's general-purpose container: an ordered sequence that can grow, shrink, and change after creation. Where a string holds characters, a list can hold anything — numbers, strings, booleans, even other lists. Lists keep insertion order, allow duplicates, and — unlike strings — are mutable: you can replace an element in place.

Theory & Concepts

Three Properties to Remember

  • Ordered — elements keep the position you gave them; index 0 is always the first item added (unless you reorder).
  • Mutable — assignment like items[2] = new_value changes the list in place. This is the big difference from strings and tuples.
  • Heterogeneous — mixed types are allowed, though in data work lists usually hold one consistent type per list.

Indexing Works Like Strings

Everything you learned about string indexing transfers directly: zero-based positions, negative indices from the right, IndexError when out of range, and len() for the element count.

Syntax Reference

Create a list with square brackets and commas. An empty pair [] starts a list you will fill later.

Creating lists — [ ] literals
daily_sales = [1250.00, 980.50, 1430.75, 1100.00]   # list of floats
skus = ["SKU-001", "SKU-002", "SKU-003"]           # list of strings
empty_batch = []                                     # empty, to be filled

print(daily_sales)     # Output: [1250.0, 980.5, 1430.75, 1100.0]
print(type(skus))      # Output: <class 'list'>

Access elements with zero-based indices; negative indices count from the end, so [-1] is always the most recent item.

Indexing — accessing elements
daily_sales = [1250.00, 980.50, 1430.75, 1100.00]

print(daily_sales[0])     # Output: 1250.0 — first element
print(daily_sales[2])     # Output: 1430.75
print(daily_sales[-1])    # Output: 1100.0 — last element

# print(daily_sales[4])   <-- IndexError: list index out of range

Because lists are mutable, you can overwrite any position directly — for example, correcting a mistyped point-of-sale entry.

Mutability — updating an element in place
daily_sales = [1250.00, 980.50, 1430.75, 1100.00]

daily_sales[1] = 995.00        # correct the POS entry at index 1

print(daily_sales)
# Output: [1250.0, 995.0, 1430.75, 1100.0]

len() returns the number of elements. Combined with sum(), it gives you an average in one line.

len() — how many elements
daily_sales = [1250.00, 995.00, 1430.75, 1100.00]

print(len(daily_sales))     # Output: 4

average = sum(daily_sales) / len(daily_sales)
print(f"Avg daily revenue: ${average:,.2f}")
# Output: Avg daily revenue: $1,193.94

Lists can contain other lists. Index twice to reach the inner values — first the row, then the column.

Nested lists — rows and columns
# Units sold per store: each inner list is one store's [weekday, weekend]
sales_grid = [[120, 85], [95, 140], [110, 90]]

print(sales_grid[1])        # Output: [95, 140] — the second store's row
print(sales_grid[1][0])     # Output: 95 — weekday units for that store
print(sales_grid[2][1])     # Output: 90

The in operator tests membership — the quick way to check whether a value is somewhere in the list.

in operator — membership test
skus = ["SKU-001", "SKU-002", "SKU-003"]

print("SKU-002" in skus)        # Output: True
print("SKU-999" in skus)        # Output: False
print("SKU-999" not in skus)    # Output: True

Edge Cases & Gotchas

Assigning a list to a second variable does not copy it — both names point to the same object, so a change through one name shows up in the other. (Hour 20 covers proper copying.)

Gotcha — assignment is not a copy
original = [100, 200, 300]
alias = original             # both names point at ONE list

alias[0] = 999

print(original)              # Output: [999, 200, 300] — changed too!
print(alias)                 # Output: [999, 200, 300]

Real-World Application

  • ETL — scripts buffer incoming transactions rows in a list before a bulk database insert.
  • Analytics — a list of daily totals feeds sum(), len(), min(), and max() for quick summary statistics.
  • Retail — a shopping cart is naturally a list of SKUs: ordered, duplicable, and editable.

Key Takeaways

  • Lists are ordered, mutable sequences created with [...]; they allow duplicates and mixed types.
  • Indexing mirrors strings: zero-based, negative from the right, IndexError out of range.
  • lst[i] = value updates in place — the key difference from immutable strings.
  • len(), sum(), and in cover counting, totaling, and membership.
  • Plain assignment aliases a list; it never copies it.
