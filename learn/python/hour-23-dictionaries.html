Dictionaries

Subtopics: Dictionary syntax, key-value pairs, accessing values by key, mutability, adding/updating items, in operator

Go to Assignment →

Labeled Data

A list answers "what is at position 3?"; a dictionary answers "what is the price?" Dicts store key–value pairs, letting you look values up by meaningful names instead of positions. Every database row, JSON API response, and configuration file maps naturally onto a dict — which is why it is arguably the most important data structure in Python.

Theory & Concepts

  • Syntax — {key: value, ...}. Keys are usually strings, but any immutable (hashable) type works: numbers, tuples. Lists cannot be keys.
  • Unique keys — writing a key twice keeps only the last value; there is no error.
  • Ordered — since Python 3.7, dicts remember insertion order.
  • Fast — key lookup is O(1) on average, regardless of size.

Syntax Reference

Create a dict with braces and colons. Reading a value uses square brackets with the key.

Creating a dict — key: value pairs
product = {
    "product_id": "P-004821",
    "product_name": "Wireless Keyboard",
    "unit_price": 49.99,
    "stock_qty": 120,
}

print(product["product_name"])   # Output: Wireless Keyboard
print(product["unit_price"])     # Output: 49.99
print(len(product))               # Output: 4 — number of pairs

Dicts are mutable: assign to an existing key to update it, or to a new key to add a pair. The same bracket syntax does both.

Updating and adding — bracket assignment
product = {"product_name": "Wireless Keyboard", "unit_price": 49.99}

product["unit_price"] = 44.99    # existing key — value updated
product["discount"] = 0.10       # new key — pair added

print(product)
# Output: {'product_name': 'Wireless Keyboard', 'unit_price': 44.99, 'discount': 0.1}

Reading a key that does not exist raises KeyError. Test membership first with in — which checks keys, not values.

in operator — does this key exist?
txn = {"transaction_id": "TXN-8842", "amount": 1500.00}

print("amount" in txn)          # Output: True
print("merchant_id" in txn)     # Output: False
print(1500.00 in txn)           # Output: False — in checks KEYS, not values

# print(txn["merchant_id"])     <-- KeyError: 'merchant_id'

Deleting a pair uses the del statement with the key.

del — remove a key-value pair
account = {"account_id": "ACC-1001", "temp_flag": True}

del account["temp_flag"]

print(account)   # Output: {'account_id': 'ACC-1001'}

Looping over a dict yields its keys; index into the dict to reach each value.

Looping — iterate keys and values
prices = {"Mouse": 29.99, "Keyboard": 49.99, "Monitor": 199.00}

for name in prices:
    print(f"{name}: ${prices[name]}")

# Output:
# Mouse: $29.99
# Keyboard: $49.99
# Monitor: $199.0

Edge Cases & Gotchas

Duplicate keys in a literal do not raise an error — the last occurrence silently wins. Watch for this when assembling dicts from generated data.

Gotcha — duplicate keys: last one wins
settings = {"retries": 3, "retries": 5}    # same key twice

print(settings)   # Output: {'retries': 5} — the 3 is silently gone

Only immutable types can be keys. A tuple key works; a list key raises TypeError because lists are unhashable.

Gotcha — keys must be hashable
daily_sales = {("S1", "2026-07-16"): 4500.00}    # tuple key — fine

print(daily_sales[("S1", "2026-07-16")])         # Output: 4500.0

# bad = {["S1", "2026-07-16"]: 4500.00}          <-- TypeError: unhashable type: 'list'

Real-World Application

  • ETL — csv.DictReader yields one dict per row, keyed by column names.
  • APIs — JSON responses deserialize directly into dicts (and lists of dicts).
  • Aggregation — (store_id, date) tuple keys map to daily totals for fast lookups.

Key Takeaways

  • Dicts map unique, hashable keys to values with {key: value} syntax; lookup is O(1).
  • Bracket assignment updates existing keys or adds new ones; del removes pairs.
  • Reading a missing key raises KeyError — check with in (which tests keys, not values).
  • Duplicate keys silently keep only the last value.
  • Dicts are the natural shape of database rows, JSON objects, and configs.
