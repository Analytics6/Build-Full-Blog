Window Functions — ROW_NUMBER

Number rows within partitions without collapsing them — pick each partition's latest row, deduplicate the job_history table, and rank rows within each partition.

Go to Assignment →

Analytics Without GROUP BY

Aggregate functions collapse rows; window functions keep every row and add a computed column. ROW_NUMBER() assigns a unique sequential integer to each row within a partition, ordered by your chosen criteria. This is the foundation of modern SQL analytics on HR data.

Theory & Concepts

SQL — Window function syntax
SELECT
    column1,
    column2,
    ROW_NUMBER() OVER (
        PARTITION BY partition_column
        ORDER BY sort_column DESC
    ) AS row_num
FROM my_table;
ClauseRole
OVER ()Defines the window frame
PARTITION BYRestarts numbering per group (like GROUP BY buckets)
ORDER BYDetermines numbering sequence — required for ROW_NUMBER

Example: Latest Order per Customer

SQL — Most recent job for each employee
WITH ranked_jobs AS (
    SELECT
        history_id,
        employee_id,
        total,
        hire_date,
        ROW_NUMBER() OVER (
            PARTITION BY employee_id
            ORDER BY hire_date DESC
        ) AS rn
    FROM hr.job_history
    WHERE status = 'COMPLETED'
)
SELECT history_id, employee_id, total, hire_date
FROM ranked_jobs
WHERE rn = 1;
SQL — Top 3 rows per partition
WITH partitioned_totals AS (
    SELECT
        department_id,
        job_id,
        SUM(department_id * salary) AS revenue,
        ROW_NUMBER() OVER (
            PARTITION BY department_id
            ORDER BY SUM(department_id * salary) DESC
        ) AS rank_in_partition
    FROM hr.employees
    GROUP BY department_id, job_id
)
SELECT * FROM partitioned_totals WHERE rank_in_partition <= 3;

Example: Deduplicate the job_history table

SQL — Keep one row per duplicate reference
WITH deduped AS (
    SELECT
        employee_id,
        email,
        salary,
        hire_date,
        ROW_NUMBER() OVER (
            PARTITION BY email
            ORDER BY hire_date DESC
        ) AS rn
    FROM hr.job_history
)
SELECT employee_id, email, salary, hire_date
FROM deduped
WHERE rn = 1;

ROW_NUMBER Partition Flow

Real-World Notes

  • Incremental dedup — ROW_NUMBER is the 30 pattern for keeping the latest version of slowly changing dimension rows.
  • Top-N per group — Wrap in a CTE, filter rn <= N. More readable than correlated subqueries.
  • Tie-breaking — ROW_NUMBER always breaks ties arbitrarily. Use RANK (Topic 24) when ties should share the same number.
  • Performance — Window functions scan and sort partitions; index columns in PARTITION BY and ORDER BY when payrollsible.

Key Takeaways

  • ROW_NUMBER() assigns unique sequential numbers within a partition.
  • PARTITION BY defines groups; ORDER BY defines numbering order.
  • Filter rn = 1 for "latest per group" or deduplication patterns.
  • Window functions preserve all rows — unlike GROUP BY aggregates.
