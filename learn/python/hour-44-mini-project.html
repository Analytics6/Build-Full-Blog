Mini Project: End-to-End Sales Analytics Pipeline

Put your sprint skills together: load a sample CSV, clean missing values, filter relevant records, aggregate with groupby, and export a summary report. This mirrors a real data analytics workflow — from raw store_sales ingest to stakeholder-ready output.

Theory & Concepts

ETL Stages

  1. Extract — pd.read_csv()
  2. Transform — clean, filter, groupby().agg()
  3. Load — df.to_csv() export

groupby()

Split by a column (e.g., store_id), apply aggregations (sum, mean, count), combine into a summary table.

ETL Pipeline Steps

Syntax Reference

Stage 1 — Extract: load the raw file and check its size before touching anything.

Extract — load the raw CSV
import pandas as pd

df = pd.read_csv("store_sales.csv")

print(df.shape)    # Output (example): (5200, 6) — raw row count

Stage 2 — Clean: drop rows with missing amounts and remove refunds or bad records with non-positive values.

Clean — drop nulls and invalid rows
df = df.dropna(subset=["amount"])
df = df[df["amount"] > 0]

print(df.shape)    # Output (example): (5108, 6) — after cleaning

Stage 3 — Aggregate: groupby() splits the rows by store, agg() computes several statistics per group in one call.

groupby().agg() — summary per store
summary = df.groupby("store_id")["amount"].agg(["sum", "mean", "count"])
summary.columns = ["total_revenue", "avg_sale", "txn_count"]

print(summary.head(2))
# Output (example):
#           total_revenue  avg_sale  txn_count
# store_id
# S-01           84210.50    412.80        204
# S-02           61055.25    377.50        162

Stage 4 — Load: export the summary to a new CSV for stakeholders.

to_csv() — export the report
summary.to_csv("store_summary_report.csv")

print("Report saved")    # Output: Report saved

Real-World Application

  • Daily summary reports — Aggregate sales by store; email CSV to regional managers.
  • Data summaries — Group transactions by account type; compute total volume.
  • dashboards — Weekly store revenue summary feeds the analytics API.
