Subtopics: String creation, single/double/triple quotes, string concatenation, replication, len()

Go to Assignment →

Text Is Everywhere

Names, addresses, product codes, log messages, file paths — a huge share of the data you will ever process is text, and in Python text lives in strings. A string is an ordered sequence of Unicode characters. This hour covers every way to create a string, why strings cannot be modified in place, and the basic operations that combine, repeat, and measure them.

Creating Strings

Single quotes '...' and double quotes "..." are completely interchangeable. Pick one style and stay consistent; switch to the other when the text itself contains a quote character.

Quotes — single and double are equivalent
product_a = 'Wireless Mouse'          # single quotes
product_b = "Wireless Keyboard"       # double quotes — same type

print(type(product_a))    # Output: <class 'str'>
print(type(product_b))    # Output: <class 'str'>

When the text contains an apostrophe or a quotation mark, wrap it in the other quote style — or escape the character with a backslash.

Quotes inside quotes — mixing styles
note_1 = "Customer's preferred branch: Downtown"   # ' inside " "
note_2 = 'She said "deliver by Friday"'             # " inside ' '
note_3 = 'Customer\'s order shipped'                # or escape with \

print(note_1)   # Output: Customer's preferred branch: Downtown
print(note_2)   # Output: She said "deliver by Friday"

Triple quotes """...""" create strings that span multiple lines, preserving the line breaks exactly as written. They also serve as docstrings.

Triple quotes — multiline strings
product_description = """Wireless Bluetooth Mouse
SKU: ELEC-4421 — in stock at 12 warehouses.
Unit price: $29.99."""

print(product_description)
# Output:
# Wireless Bluetooth Mouse
# SKU: ELEC-4421 — in stock at 12 warehouses.
# Unit price: $29.99.

Escape Sequences & Raw Strings

A backslash gives the next character special meaning: \n is a newline, \t a tab, \\ a literal backslash.

Escape sequences — \n and \t
receipt = "Item:\tWireless Mouse\nPrice:\t$29.99"

print(receipt)
# Output:
# Item:   Wireless Mouse
# Price:  $29.99

Raw strings (prefix r) turn escapes off, keeping every backslash literal. They are essential for Windows file paths — without the prefix, \t in a path silently becomes a tab character.

Raw strings — r"..." for file paths
inventory_path = r"C:\etl\retail\inventory_2026.csv"   # backslashes kept
data_path = "C:\\etl\\banking\\transactions.csv"       # or escape each one

print(inventory_path)   # Output: C:\etl\retail\inventory_2026.csv
print(data_path)        # Output: C:\etl\banking\transactions.csv

Combining & Measuring Strings

The + operator joins strings end to end. It only joins strings with strings — mixing in a number raises TypeError unless you cast with str().

Concatenation — joining with +
first_name = "Priya"
last_name = "Sharma"

full_name = first_name + " " + last_name
print(full_name)   # Output: Priya Sharma

order_id = 1001
label = "Order #" + str(order_id)      # cast the number first
print(label)       # Output: Order #1001

The * operator repeats a string a whole number of times — handy for separators and simple text bars in console reports.

Replication — repeating with *
divider = "-" * 30
print(divider)          # Output: ------------------------------

print("SKU-" * 3)       # Output: SKU-SKU-SKU-
print("=" * 10)         # Output: ==========

len() returns the number of characters in a string, counting spaces and punctuation. It is one of the most used functions in all of Python.

len() — string length
account_code = "ACC-8842-01"

print(len(account_code))     # Output: 11
print(len(""))               # Output: 0 — the empty string
print(len("a b"))            # Output: 3 — the space counts too

Edge Cases & Gotchas

Strings are immutable: once created, they can never be changed. Every operation that seems to modify a string actually builds a new one, which is why method results must be assigned back to a variable.

Gotcha — strings are immutable
account_code = "ACC-8842"

# account_code[0] = "B"     <-- TypeError: 'str' object does not support item assignment

# "Changing" a string really means building a new one
account_code = "B" + account_code[1:]
print(account_code)           # Output: BCC-8842

Real-World Application

  • ETL — raw strings hold Windows paths to store_sales and order_items CSV files without escape bugs.
  • Reporting — concatenated name fields feed customer-facing letters; "-" * 40 draws separators in console reports.
  • Validation — len() checks that account codes and phone numbers have the expected number of characters.

Key Takeaways

  • Single and double quotes are equivalent; triple quotes handle multiline text.
  • Use the r prefix for file paths so backslashes stay literal.
  • + joins, * repeats, len() measures.
  • Strings are immutable — every "change" creates a new string, so assign results back.
  • Numbers must be cast with str() before concatenation.
