COALESCE and NULLIF

Handle NULL values in expressions — replace missing data with defaults and prevent divide-by-zero errors in SQL expressions.

Go to Assignment →

Why NULL Handling Matters

In relational databases, NULL means unknown or not provided — not zero, not an empty string. A row may have no optional middle name; a foreign key may be unset; a numeric column may be unknown. COALESCE and NULLIF let you write safe, readable expressions that fill gaps and avoid silent calculation errors.

Theory & Concepts

FunctionPurposeReturns
COALESCE(a, b, c, ...)First non-NULL argumentFirst value that is not NULL, or NULL if all are NULL
NULLIF(a, b)Convert matching values to NULLNULL when a = b; otherwise returns a

Remember: NULL = NULL is unknown, not true. Use IS NULL / IS NOT NULL for comparisons. COALESCE is variadic — you can chain many fallbacks. NULLIF is commonly paired with division to turn zero denominators into NULL instead of errors.

COALESCE in HR Reports

SQL — Effective value with COALESCE
-- Use min_salary when set; otherwise fall back to salary
SELECT
    job_id,
    job_title,
    COALESCE(min_salary, salary) AS effective_price
FROM hr.jobs
WHERE category = 'Sales';
SQL — Customer display name with fallbacks
SELECT
    employee_id,
    COALESCE(
        NULLIF(TRIM(middle_name), ''),
        first_name || ' ' || last_name
    ) AS display_name,
    COALESCE(email, Manager, 'no contact on file') AS contact
FROM hr.employees;

NULLIF in HR Calculations

SQL — Safe average transaction salary
-- Avoid divide-by-zero when a branch has no hr.job_history
SELECT
    b.branch_name,
    SUM(t.salary) AS total_volume,
    COUNT(t.employee_id) AS history_count,
    SUM(t.salary) / NULLIF(COUNT(t.employee_id), 0) AS avg_txn
FROM hr.departments b
LEFT JOIN hr.job_history t ON t.department_id = b.department_id
GROUP BY b.department_id, b.branch_name;

Real-World Notes

  • ETL staging — Use COALESCE(updated_at, created_at) as a reliable watermark column when syncing the job_history table incrementally.
  • Data quality — NULLIF(column, '') converts empty strings from legacy CSV feeds into proper NULLs before loading into the employees table.
  • Reporting — Dashboards should never show blank KPI cells; COALESCE(metric, 0) gives analysts a zero instead of a missing value — but document when zero and NULL mean different things.
  • HR compliance — Do not use COALESCE to mask missing KYC fields; flag NULLs for review instead of substituting defaults.

Common Mistakes

  • Using WHERE column = NULL — always use IS NULL.
  • Assuming COALESCE(a, 0) means the value was zero; it may have been unknown.
  • Chaining too many fallbacks without documenting business priority order.

Key Takeaways

  • COALESCE returns the first non-NULL value from a list of expressions.
  • NULLIF(a, b) returns NULL when a equals b — ideal for safe division.
  • NULL is not zero or empty string; treat it as unknown in HR data.
  • Use these functions in SELECT, WHERE, JOIN, and aggregate expressions.
