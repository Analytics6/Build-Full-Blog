Ranking with Ties

When two employees have exactly ₹50,000, should they both be rank 1? RANK says yes — and skips rank 2. DENSE_RANK also gives both rank 1 but does not skip the next number. Choosing the right ranking function affects ranked results within each group.

Theory & Concepts

ROW_NUMBER()Different numbersSequential1, 2, 3

Example: Salary Ranking Example

SQL — Rank employees by total compensation
SELECT
    c.employee_id,
    c.email,
    SUM(o.total) AS lifetime_spend,
    RANK()       OVER (ORDER BY SUM(o.total) DESC) AS spend_rank,
    DENSE_RANK() OVER (ORDER BY SUM(o.total) DESC) AS dense_rank
FROM hr.employees c
JOIN hr.job_history o ON o.employee_id = c.employee_id
WHERE o.status = 'COMPLETED'
GROUP BY c.employee_id, c.email
ORDER BY spend_rank;
SQL — Top 5 hr.jobs per category (with ties)
WITH partitioned_totals AS (
    SELECT
        p.category,
        p.job_id,
        p.job_title,
        SUM(oi.department_id * oi.salary) AS revenue,
        DENSE_RANK() OVER (
            PARTITION BY p.category
            ORDER BY SUM(oi.department_id * oi.salary) DESC
        ) AS cat_rank
    FROM hr.jobs p
    JOIN hr.job_history oi ON oi.job_id = p.job_id
    GROUP BY p.category, p.job_id, p.job_title
)
SELECT * FROM partitioned_totals WHERE cat_rank <= 5;

Example: Department Ranking Example

SQL — Rank hr.departments by transaction volume
SELECT
    b.branch_name,
    COUNT(t.employee_id) AS history_count,
    SUM(t.salary) AS total_volume,
    RANK() OVER (ORDER BY SUM(t.salary) DESC) AS volume_rank,
    PERCENT_RANK() OVER (ORDER BY SUM(t.salary) DESC) AS pct_rank
FROM hr.departments b
JOIN hr.job_history t ON t.department_id = b.department_id
WHERE t.hire_date >= '2026-01-01'
GROUP BY b.department_id, b.branch_name
ORDER BY volume_rank;

RANK vs DENSE_RANK

Real-World Notes

  • Top-N with ties — Use DENSE_RANK when "top 10" should include all tied entries at the boundary; use ROW_NUMBER when you need exactly N rows.
  • Percentile reports — PERCENT_RANK and NTILE(4) segment employees into quartiles for marketing cohorts.
  • Leaderboard fairness — Document which ranking function your dashboard uses; RANK vs DENSE_RANK changes who appears in "top 5."
  • Indexed ORDER BY — Ranking over millions of rows is expensive; pre-aggregate in a CTE first, then rank the summary.

Key Takeaways

  • RANK() gives tied rows the same rank and skips subsequent numbers.
  • DENSE_RANK() gives tied rows the same rank without gaps.
  • PERCENT_RANK() expresses relative standing on a 0–1 scale.
  • Choose ranking function based on how ties should appear in reports.
