String Indexing

Subtopics: Positive indexing, negative indexing, accessing individual characters

Go to Assignment →

Reaching Into a String

A string is an ordered sequence, which means every character has a numbered position. Indexing with square brackets lets you pull out any single character — the first letter of a code, the last digit of an account number. The one rule beginners must internalize: counting starts at 0, not 1.

Theory & Concepts

Positive and Negative Indices

Positive indices count from the left starting at 0; negative indices count from the right starting at -1. For the string "PYTHON":

CharacterPYTHON
Positive index012345
Negative index-6-5-4-3-2-1

Negative indexing shines when you care about the end of a string regardless of its length: s[-1] is always the last character, whether the string has 5 characters or 500.

Rules and Pitfalls

  • len(s) returns the character count; the largest valid index is len(s) - 1, a classic off-by-one trap.
  • An out-of-range index raises IndexError: string index out of range.
  • Strings are immutable — s[0] = 'X' raises TypeError. Build a new string instead.
  • Indexing an empty string always fails: even ""[0] is an IndexError. Check len() or truthiness first.

Syntax Reference

Example — Indexing an account number
# Inspect individual characters of a banking account code
account = "ACC-8842-01"

print(account[0])     # Output: A — first character
print(account[4])     # Output: 8 — start of the numeric part
print(account[-1])    # Output: 1 — last character (branch digit)
print(account[-2])    # Output: 0 — second from the right
print(len(account))   # Output: 11

# The largest valid index is len - 1:
print(account[10])    # Output: 1 — same as account[-1]
# print(account[11])  <-- IndexError: string index out of range
Example — Routing log lines by first character
# Classify pipeline log lines by their severity prefix
log_line = "ERROR: Failed to load store_sales.csv"

if log_line[0] == "E":
    print("Critical pipeline failure — page the on-call engineer")
elif log_line[0] == "W":
    print("Warning — review the ETL step tomorrow")
else:
    print("Informational — no action needed")

# Output: Critical pipeline failure — page the on-call engineer

Because strings are immutable, you cannot overwrite a character in place — instead, build a new string from the pieces you want to keep.

Example — Immutability workaround
sku = "P-4421"

# sku[0] = "S"                 <-- TypeError: strings are immutable
sku = "S" + sku[1:]              # new string: keep everything after index 0
print(sku)                       # Output: S-4421

Indexing an empty string always raises IndexError — even position 0 does not exist. Guard with a truthiness check before touching the first character.

Example — Guarding against empty strings
customer_note = ""

if customer_note:                # empty string is falsy — block is skipped
    print(customer_note[0])
else:
    print("No note on file")

# Output: No note on file

Real-World Application

  • Routing — the first character of a transaction_id decides which processing queue a record enters.
  • Validation — SKU conventions like "first char P = product, S = service" are enforced with a single index check.
  • Check digits — card and account numbers store a checksum in the last position, read with [-1].

Key Takeaways

  • Indexing starts at 0; the last valid index is len(s) - 1.
  • Negative indices count from the right — s[-1] is always the last character.
  • Out-of-range access raises IndexError; empty strings cannot be indexed at all.
  • Strings are immutable — rebuild with concatenation and slicing instead of assigning.
