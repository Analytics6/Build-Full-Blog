Nested Data Structures

Subtopics: List of dictionaries, dictionary of lists, dictionary of dictionaries, deep indexing

Go to Assignment →

Real Data Is Nested

Flat lists and dicts model simple data, but the payloads you meet in practice — JSON API responses, configuration files, report structures — are containers inside containers: a list of order dicts, a dict mapping branches to lists of accounts. The skill of this hour is deep indexing: chaining [key] and [index] steps from the outermost container inward, one hop at a time.

Theory & Concepts

The Three Common Shapes

ShapeModelsExample
List of dictsTable rowsOrders, transactions, CSV records
Dict of listsGrouped dataAccounts grouped by branch
Dict of dictsKeyed recordsProducts indexed by SKU

How to read a deep index: evaluate left to right. data["stores"][1]["daily_sales"][0] means: take data, get its "stores" list, take element 1, get that dict's "daily_sales" list, take element 0. When lost, print each intermediate step.

Syntax Reference

A list of dicts is Python's version of a database table: each dict is one row with named fields.

List of dicts — table rows
orders = [
    {"order_id": "ORD-101", "items": ["SKU-1", "SKU-2"], "total": 1500},
    {"order_id": "ORD-102", "items": ["SKU-3"], "total": 450},
]

print(orders[0]["order_id"])    # Output: ORD-101 — row 0, column order_id
print(orders[0]["items"][1])    # Output: SKU-2 — second item of the first order
print(orders[1]["total"])       # Output: 450

A dict of lists groups records under a key — here, accounts per branch.

Dict of lists — grouped records
branches = {
    "Mumbai": [{"account_id": "ACC-1", "balance": 5000}],
    "Delhi":  [{"account_id": "ACC-2", "balance": 3200}],
}

print(branches["Delhi"][0]["balance"])   # Output: 3200
print(len(branches["Mumbai"]))            # Output: 1 — accounts in Mumbai

A dict of dicts gives each record a lookup key — like a table with a primary key.

Dict of dicts — records by key
catalog = {
    "SKU-1": {"name": "Mouse", "price": 29.99},
    "SKU-2": {"name": "Keyboard", "price": 49.99},
}

print(catalog["SKU-2"]["price"])    # Output: 49.99 — straight to the record
print(catalog["SKU-1"]["name"])     # Output: Mouse

Deeper structures combine all three shapes. Read the chain hop by hop — and loop the outer container to process every inner record.

Deep indexing — a realistic nested payload
region_sales = {
    "region": "North",
    "stores": [
        {"id": "STR-001", "daily_sales": [8500, 9200, 7800]},
        {"id": "STR-002", "daily_sales": [9200, 8800, 9500]},
    ],
}

# Hop by hop: dict -> list -> dict -> list -> value
value = region_sales["stores"][1]["daily_sales"][0]
print(value)     # Output: 9200 — store 2, day 1

# Loop the outer list; compute per-store averages from the inner lists
for store in region_sales["stores"]:
    avg = sum(store["daily_sales"]) / len(store["daily_sales"])
    print(f"{store['id']}: {avg:.1f}")

# Output:
# STR-001: 8500.0
# STR-002: 9166.7

Edge Cases & Gotchas

One missing key or short list anywhere along a chain crashes the whole expression. Chain .get() with default containers to navigate defensively when data may be incomplete.

Gotcha — safe navigation with get() chains
payload = {"customer": {"name": "Priya"}}    # no "address" key at all

# payload["customer"]["address"]["city"]     <-- KeyError: 'address'

city = payload.get("customer", {}).get("address", {}).get("city", "UNKNOWN")
print(city)     # Output: UNKNOWN — no crash, each hop falls back to {}

Real-World Application

  • APIs — payment and CRM responses arrive as nested JSON; deep indexing extracts the fields your tables need.
  • Reports — dict-of-lists groupings power per-branch and per-region summaries.
  • ETL — flattening a list of nested orders into one row per order item is a daily transformation task.

Key Takeaways

  • Nested structures combine lists and dicts; read index chains left to right, one hop at a time.
  • List of dicts = table rows; dict of lists = grouped data; dict of dicts = keyed records.
  • Loop the outer container and access inner fields to process every record.
  • Chained .get(..., {}) calls navigate incomplete data without crashing.
  • Nested JSON handling is a core ETL skill — this structure is the API world.
