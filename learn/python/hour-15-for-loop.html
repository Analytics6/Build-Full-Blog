For Loop

Subtopics: for loop syntax, range() function (start, stop, step), iterating over strings

Go to Assignment →

Do This for Every Item

The for loop is the workhorse of repetition: process every row in a file, every transaction in a batch, every character in a code. Python's for iterates directly over the elements of any sequence — no index bookkeeping required. Use it whenever the number of iterations is known or determined by a collection; the next hour's while loop covers the open-ended case.

Theory & Concepts

The Loop Variable

for item in sequence: binds item to each element in turn and runs the indented body once per element. The loop variable is a real variable — name it meaningfully (price, txn, row), not i, unless it truly is just a counter.

range()

range(start, stop, step) generates integers from start up to but not including stop. One argument means range(stop) starting at 0; three arguments add a step. Ranges are lazy — they produce numbers on demand, so range(1_000_000) costs almost nothing.

The Accumulator Pattern

Initialize a total before the loop, add to it inside the loop, use it after. This pattern underlies nearly every aggregation you will ever write.

Syntax Reference

Iterating a list directly: the loop variable receives each element, in order.

for over a list — process each element
prices = [29.99, 15.50, 42.00]

for price in prices:
    print(f"Line item: ${price}")

# Output:
# Line item: $29.99
# Line item: $15.5
# Line item: $42.0

The accumulator pattern: a running total built one element at a time.

Accumulator pattern — summing revenue
prices = [29.99, 15.50, 42.00, 8.75]
total_revenue = 0                        # initialize BEFORE the loop

for price in prices:
    total_revenue += price               # accumulate INSIDE the loop

print(f"Daily revenue: ${total_revenue:.2f}")
# Output: Daily revenue: $96.24

range() drives counted loops. Remember the stop value is excluded — range(1, 6) ends at 5.

range() — counted iteration
for day in range(1, 6):
    print(f"Generating report for day {day}")

# Output:
# Generating report for day 1
# Generating report for day 2
# Generating report for day 3
# Generating report for day 4
# Generating report for day 5

A step value processes work in strides — here, batches of 1000 transactions.

range() with step — batch processing
for batch_start in range(0, 5000, 1000):
    print(f"Processing transactions {batch_start} to {batch_start + 999}")

# Output:
# Processing transactions 0 to 999
# Processing transactions 1000 to 1999
# Processing transactions 2000 to 2999
# Processing transactions 3000 to 3999
# Processing transactions 4000 to 4999

Strings are sequences too — a for loop visits each character, useful for validating codes character by character.

for over a string — character by character
sku = "AB12"
digit_count = 0

for ch in sku:
    if ch.isdigit():
        digit_count += 1

print(f"{digit_count} digits in {sku}")   # Output: 2 digits in AB12

Edge Cases & Gotchas

Modifying a list while looping over it skips elements, because the loop's internal position no longer matches the shifted list. Iterate over a copy instead.

Gotcha — never mutate the list you are looping
amounts = [100, -50, 200, -75, 300]

for amt in amounts[:]:          # loop over a COPY ([:])...
    if amt < 0:
        amounts.remove(amt)      # ...so removing from the original is safe

print(amounts)                   # Output: [100, 200, 300]

# Looping over `amounts` directly while removing would skip -75.

Real-World Application

  • ETL — loop over the files in a folder, transforming and loading each one.
  • Aggregation — the accumulator pattern computes daily totals from transaction amounts.
  • Batching — range(0, total, batch_size) chunks big workloads into database-friendly pieces.

Key Takeaways

  • for item in sequence: visits each element — no index management needed.
  • range(start, stop, step) excludes the stop value; one argument starts from 0.
  • The accumulator pattern (init before, add inside, use after) powers every aggregation.
  • Strings iterate character by character.
  • Never modify a list while looping over it — iterate a copy with [:].
